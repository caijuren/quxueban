# 趣学伴 1.7.5 下一轮改进清单

更新时间：2026-04-30

## 目标

1.7.5 作为进入 1.8 前的稳定汇总版，优先处理低风险、高收益的稳定性和使用体验问题。

## 优先级 P0

- 正式环境完成 1.7.4 migration 验证。
- 正式环境执行《封神演义》状态修复脚本，并确认图书馆展示为最近阅读记录排序。
- 回归今日概览学习时长与钉钉推送口径。
- 确认 2026 节假日排程在五一周、国庆周不会发布假期任务。

## 优先级 P1

- 清理后端核心链路里的调试 `console.log`，尤其：
  - `tasks.ts` 发布计划日志
  - `plans.ts` 今日任务过滤日志
  - `dingtalk.ts` 推送详情日志
  - `settings.ts` 设置读写日志
- 把节假日表维护流程写入部署检查清单。
- 增加一个节假日排程单元测试，覆盖五一周和国庆周。
- 增加今日概览学习时长测试，覆盖：
  - 已完成实际时长
  - 部分完成实际时长
  - 实际时长为 0
  - 未填写实际时长回退预计时长

## 优先级 P2

- 梳理 mock mode 分支，明确哪些只用于测试，哪些可以后续隔离。
- 图书馆封面展示继续微调：为横版/方形封面补统一留白策略。
- 图书详情页阅读记录弹窗继续收口，减少手写表单样式重复。
- 统一前后端节假日表来源，后续可考虑由后端 `/api/config/holidays` 输出给前端，减少双表维护。

## 暂不进入 1.7.5

- 1.8 能力模型数据回流。
- AI 自动推荐任务。
- 自动联网抓取节假日。
- 多应用服务器目录迁移。
- CI/CD 大改。
