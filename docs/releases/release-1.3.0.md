# 趣学伴 1.3 发布计划

更新时间：2026-04-25

## 版本定位

`1.3` 是 1.2 封板后的稳定与体验提升版本。

核心目标只有两个：

1. 所有现有功能完善，并通过完整测试
2. UI 做一次全面提升，形成统一、稳定、可持续迭代的产品体验

---

## 一、功能完善与稳定

### 1. P0 核心流程回归

- 登录、退出、登录状态保持正常
- 当前孩子显示、切换、刷新后恢复正常
- 首页数据跟随当前孩子正常刷新
- 任务管理列表正常加载、筛选、编辑、删除
- 任务详情编辑后，任务列表与首页数据同步更新
- 学习计划创建、发布、查看、调整正常
- 今日任务打卡、取消、同步到首页正常
- 设置页孩子管理正常新增、编辑、删除孩子，不再要求 PIN/密码

### 2. 学习与阅读体系回归

- 图书馆正常加载当前孩子图书
- 图书馆新增、编辑、删除、筛选、年龄筛选正常
- 图书详情正常打开，阅读状态、评分、收藏正常
- 阅读中心正常展示在读图书和阅读进度
- 阅读记录新增后，图书馆、阅读页、统计页同步正常

### 3. 数据与报告回归

- 学习统计只跟随左侧当前孩子
- 学习统计核心指标、趋势图正常
- 学习报告只跟随左侧当前孩子
- 学习报告列表、生成、收藏、删除正常
- 成就页正常加载，不能因为缺接口或缺数据导致页面崩溃

### 4. 后端与数据库稳定

- 后端健康检查正常
- PM2 服务重启后正常恢复
- Prisma migration status 保持 clean
- 生产数据库关键表和字段与当前代码匹配
- 后端日志中没有持续出现的 P2022、P3009、P3018 等迁移/字段错误
- 不再使用旧部署脚本，不再手动误跑 `migrate reset`

### 5. 1.3 功能验收输出

- 完成一份功能回归记录
- 每个 P0 流程至少人工验证一次
- 发现的问题按 P0/P1/P2 分类修复
- P0 全部清零后再进入 UI 全面提升

---

## 二、UI 全面提升

### 1. 全局体验统一

- 左侧导航、顶部区域、页面标题统一
- 页面间距、字号、卡片圆角、阴影、边框统一
- 按钮、表单、筛选器、弹窗、空状态、加载态统一
- 去掉临时感、模板感、重复说明文字
- 页面文案更像真实产品，而不是功能说明文档

### 2. 核心页面提升

- 首页：信息优先级清晰，适合每天打开使用
- 任务管理：列表、详情、编辑、发布路径更顺
- 学习计划：周计划结构更清楚，发布状态更明确
- 图书馆：书架感更强，筛选和操作更顺手
- 阅读中心：聚焦在读、进度、记录和下一步行动
- 学习统计：减少噪音，突出趋势和结论
- 学习报告：更像报告中心，而不是表单页面
- 设置页：保留核心项，减少分散入口

### 3. 响应式与可用性

- 桌面端主流程不拥挤、不跳动
- 窄屏和移动端核心页面可用
- 长文本、按钮、卡片内容不溢出
- 加载态和空状态不造成页面闪烁
- 错误提示明确、可恢复

### 4. 视觉质量标准

- 不做大而空的营销页
- 不使用无意义装饰背景
- 不新增没有业务价值的大卡片
- 不让页面变成单一色调
- 优先清晰、稳定、可反复使用的家庭学习工具体验

---

## 三、1.3 不包含

- 新的大型 AI 能力
- 新的复杂孩子端
- 多家庭商业化后台
- 大规模数据迁移重构
- 完整 2.0 能力模型

这些内容进入 1.5 或 2.0。

---

## 四、建议执行顺序

1. 功能回归测试
2. 修复 P0/P1 功能问题
3. 后端日志与数据库状态确认
4. UI 统一规范整理
5. 按页面逐个提升 UI
6. 全站回归
7. 1.3 封板

---

## 五、1.3 封板标准

- P0 功能问题清零
- 核心页面人工回归通过
- 前后端构建通过
- 线上发布流程稳定可重复
- 后端健康检查通过
- 数据库 migration 状态正常
- UI 不再呈现明显拼接、临时、模板化状态
