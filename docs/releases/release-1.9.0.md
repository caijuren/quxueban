# 趣学伴 1.9.0 发布记录

发布日期：2026-05-02

## 版本定位

`1.9.0` 是三层准备度模型首轮闭环版。目标是把家长端的任务、目标、能力模型和今日记录统一到“交付层、认知层、稳定性层”的信息架构里。

## 本次变更

- 新增三层准备度配置 `frontend/src/lib/readiness-model.ts`。
- 今日概览收口为“今日状态总览”，横向展示交付层、认知层和稳定性层证据摘要。
- 任务完成弹窗新增：
  - 状态记录：睡眠、情绪、外部负载。
  - 认知记录：尝试次数、是否使用提示、主要错因、复盘质量。
- 任务管理页新增三层数量摘要和按层级筛选。
- 任务卡片精简为任务名、层级能力分类、规则时长孩子和底部操作区。
- 任务新建/编辑表单重排为基础信息、三层归属、目标连接、执行设置和额外字段。
- 任务详情页改为任务档案结构，新增 1.9 回流说明和认知层采集建议。
- 目标管理接入三层归类和目标字段保存回显。
- 能力模型页主体改为三层能力树，并兼容旧 `subject / thinking / habit / health` 保存数据。
- 通用导出弹窗、发布计划弹窗和图书馆添加图书入口做了轻量主按钮样式收口；完整 UI 统一后置。
- 版本号更新到 `1.9.0`。

## 数据存储

本次不新增独立后端 schema。

- 稳定性层记录写入 `daily_checkins.metadata`。
- 数理认知试点记录写入 `daily_checkins.metadata`。
- 能力模型继续使用家庭设置里的 `abilityModel`。
- 目标继续使用家庭设置里的 `goalsByChild`。

## 暂不包含

- 独立稳定性层和认知层 schema。
- 复杂诊断引擎。
- 目标校权重和适配度。
- 置信度、Velocity、余力指数和跨学科交叉验证。
- 仪表盘重构。
- 完整 UI 弹窗和按钮统一。

## 上线口径

1.9.0 首轮可上线。上线后冻结 1.9.0 范围，后续进入 1.9.x 或单独 UI 收口版本。
