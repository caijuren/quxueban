# 趣学伴版本迭代计划

更新时间：2026-05-05

## 当前判断

当前产品已经完成 `1.9.1` UI 架构升级版上线前收口，核心能力不再停留在“任务记录工具”，而是进入“三层准备度模型 + 目标管理闭环”的阶段。

`1.9.1` 已经完成桌面端 UI 收口，下一步不能立刻扩数据体检、诊断、报告或 AI。当前最关键的问题是：三层准备度已经成为主模型，但还偏“分类框架”；目标管理已经有目标、能力点和任务关系，但还没有真正成为“目标准备工作台”。

因此，下一版只做一个主题：

```text
三层准备度内核定型
```

目标是先把三公基础模板、三层能力点、观察指标、推荐任务和数据来源定清楚，再让目标管理作为第一应用场景承接它。数据治理闭环、规则诊断增强、目标校权重、报告和 AI 全部顺延。

## 后续版本拆分

### 1.9.1：UI 架构升级版

定位：

解决当前页面混乱、信息密度过高、控件样式不统一的问题。此版本以 UI 信息架构和可用性为核心，不新增复杂业务能力。

为什么放在 1.9.1：

- `1.9.0` 已经完成三层模型和目标引擎首轮落地。
- 现在的主要矛盾是功能已经进来了，但页面承载方式不够清晰。
- 如果先继续做诊断和 AI，页面会更难收口。

范围：

- 重新梳理家长端主信息架构：今日、任务、目标、成长、阅读、设置的层级关系。
- 统一页面头部、操作区、筛选区、卡片密度、空状态、提示条、弹窗宽度和按钮层级。
- 今日概览改成更清晰的“今日行动 + 三层状态 + 风险提醒”结构。
- 任务完成弹窗重新分组，降低状态记录、认知记录、证据上传的视觉噪音。
- 任务列表、任务编辑器、任务详情统一字段优先级。
- 目标管理、能力模型、成长仪表盘统一三层准备度表达。
- 成长仪表盘 Tab 命名和边界收口：
  - `执行力分析` -> `执行诊断`，只诊断任务执行断点。
  - `学科能力` -> `能力证据`，只展示样本、证据和薄弱方向，不伪造能力分。
  - `心态兴趣` -> `稳定性与兴趣`，只观察状态、负载和启动偏好，不做心理诊断。
  - `阅读能力` -> `阅读成长`，只展示阅读投入、结构、连续性和记录质量。
- 数据体检和图书详情的异常修复入口保持清晰，不做成隐藏功能。
- 本版本只做桌面端截图级走查，移动端后续单独规划。

不做：

- 不新增 AI。
- 不做大规模后端 schema 迁移。
- 不新增新的复杂诊断规则。
- 不重做品牌视觉，只做产品内工作台式 UI 升级。

验收：

- 核心页面桌面端无明显挤压、重叠和字段噪音。
- 移动端不作为本版本验收项。
- 同类按钮、筛选、标签、提示条样式统一。
- 用户能在 10 秒内判断：今天做什么、哪里有风险、哪些数据需要补。

### 1.9.2：三层准备度内核定型版

定位：

先定三层准备度内核，再让目标管理承接它。

本版本的第一优先级是三层准备度模型定型。目标管理是三层模型的第一应用场景，同步收口但必须服从模型。生产限流导致登录和孩子数据 429 的问题作为发版前稳定性修正一并处理。

为什么放在 1.9.2：

- 三层准备度已经成为主模型，但还缺三公基础模板、能力点水位、观察指标、推荐任务和数据来源。
- 目标管理已经能承接能力点和任务，但如果没有模型约束，就会变成目标列表和任务入口。
- 如果先做数据治理或 AI，三层内核没有定型，后续建议和报告会缺少稳定锚点。

范围：

- 三层准备度页从模型说明页升级为模型配置和能力点维护页。
- 默认支持“三公基础模板”。
- 三层结构固定为：
  - 交付层：英语能力、数理逻辑、大语文、项目成果、临场应变。
  - 认知层：信息处理速度、规则内化、逻辑深度、批判思维、迁移应用。
  - 稳定性层：睡眠作息、运动健康、情绪恢复、执行完成率、延期率、复盘频率。
- 目标管理页升级为“目标准备工作台”。
- 目标支持配置目标方向、目标周期、所属层级、能力点、成功标准、支撑任务和复盘节奏。
- 目标卡片展示三层能力点、当前进度、成功标准、支撑任务数量、当前缺口和下一步动作。
- 目标缺任务、缺能力点、缺成功标准时给出明确动作。
- 能力点可以进入目标创建或任务创建。
- 生产限流修正：
  - `/api/health` 和 `/api/version` 不进入业务限流。
  - 普通业务 API 限流水位提高。
  - 登录接口保留独立较严格限流。
  - 登录限流不影响 `/api/children` 和其他已登录业务接口。

不做：

- 不做数据体检完整修复闭环。
- 不做目标校录取概率。
- 不做上实、上外、浦外适配分。
- 不做 AI 建议。
- 不做自动报告。
- 不做复杂诊断引擎。
- 不做移动端专项改版。

验收：

- 家长进入目标管理页，10 秒内能看到当前目标方向、主要缺口和下一步动作。
- 家长进入三层准备度页，10 秒内能看懂三公基础模板、三层结构和能力点定义。
- 家长能从目标卡片直接知道目标服务哪个三层能力点。
- 家长能看出哪些目标没有任务支撑。
- 家长能从三层准备度页进入目标创建或任务创建。
- 页面默认支持“三公基础模板”，但不输出录取概率、排名和目标校适配分。
- 生产环境正常页面加载不触发 `/api/login`、`/api/children`、`/api/health`、`/api/version` 的 429。

### 1.9.3：数据治理闭环版

定位：

把数据体检从“发现问题”升级为“发现问题 -> 定位问题 -> 修复问题 -> 复查”的闭环。

范围：

- 数据体检页完善任务、目标、图书、阅读记录、打卡记录的缺口检查。
- 阅读页码异常修复链路收口。
- 图书缺 ISBN、页数、类型的快速修复入口。
- 任务缺能力点、目标类型、记录方式的快速修复入口。
- 目标缺能力点、周期、成功标准的快速修复入口。
- 修复后刷新对应缓存，并能回到数据体检复查。
- 数据质量分层：阻塞 AI、影响报告、影响统计、建议补充。

不做：

- 不做批量自动改写历史数据。
- 不让系统自动猜测并覆盖家长输入。

### 1.9.4：规则诊断增强版

定位：

把规则诊断从 V0.1 提升为可解释的家庭学习诊断引擎，为 2.0 AI 提供稳定依据。

范围：

- 增强置信度：样本数量、数据来源数量、最近更新时间、证据覆盖率。
- 完善 Velocity：4 周/8 周成长斜率和稳定性波动率。
- 完善余力指数：强项维持、短板加码、过载减压。
- 增加假性退步、假性爆发、认知阻力位、输入不足、输出虚高。
- 完善跨学科交叉验证，区分单科问题和通用稳定性问题。
- 每条诊断必须展示证据、规则说明和下一步动作。

不做：

- 不接 AI 生成诊断结论。
- 不输出无法解释的黑盒分数。

### 1.9.5：目标校权重和成长分析体系版

定位：

在 1.9.2 三层准备度内核定型后，再把三公基础模板扩展为目标校权重和成长分析体系。

范围：

- 支持上实、上外、浦外方向的内部权重配置。
- 成长总览只保留目标准备信号，不直接展示学校适配分。
- 目标准备度展示证据完整度、能力缺口、稳定性风险、置信度和下一步动作。
- 初中方向预留中考、自招、特色高中、国际路线配置。
- 成长总览、执行诊断、能力证据、稳定性与兴趣、阅读成长统一使用同一套规则摘要。
- 目标缺口可跳转到任务、目标或数据体检。
- 阅读成长档案从单本书扩展到孩子维度的阅读成长总览。

不做：

- 不做录取概率。
- 不做和其他孩子排名。
- 不制造焦虑型文案。
- 不在目标未配置或证据不足时输出伪精确学校适配分。

### 1.9.6：规则版报告和学习事件日志版

定位：

完成 2.0 前最重要的数据底座：事件可追溯、报告可解释。

范围：

- 建立学习事件日志：任务创建、编辑、完成、延期，计划发布，阅读记录，目标调整等。
- 规则版周报/月报：事实、趋势、风险、建议分层展示。
- 报告建议可以转成目标草稿或任务草稿。
- 报告引用真实数据，不使用静态 mock。
- 事件日志作为后续 AI 读取的标准输入。

不做：

- 不让 AI 自动生成最终报告。
- 不自动执行报告建议。

### 2.0：AI 驱动版

定位：

2.0 不再以新增传统页面为主，而是让 AI 基于 1.9.x 沉淀的数据闭环、规则诊断和事件日志，成为家长的学习决策助手。

范围：

- AI 学习画像：优势、短板、节奏、卡点、稳定性风险。
- AI 数据缺口发现：说明哪些结论证据不足。
- AI 能力变化解释：引用任务、阅读、打卡、目标和报告数据。
- AI 目标建议：生成可编辑目标草稿。
- AI 任务推荐：生成可编辑任务草稿。
- AI 阅读建议：优先基于家里已有书推荐。
- AI 周报/月报草稿：区分事实、分析、建议。
- AI 家长问答：围绕真实数据回答问题。
- AI 设置：可关闭、可查看使用数据、可清除画像纠偏。

约束：

- AI 不自动执行关键操作。
- AI 建议必须展示依据。
- 数据不足时必须明确提示。
- 家长可否定、纠偏和关闭。
- 不输出医学、心理诊断式结论。

## 3.0 方向

3.0 不进入当前开发主线，只保留方向：

- 长期成长档案 Portfolio。
- 学期/年度归档。
- 家庭多成员协作和权限。
- 老师/机构协作入口。
- 教材、课程、分级体系库。
- 学习路径模板生态。
- 移动端或小程序。
- 高级隐私、导出、分享和审计。
