# 1.9.1 UI 架构升级走查记录

检查日期：2026-05-04

## 本轮范围

本轮先做低风险 UI 收口，不调整业务接口和数据计算逻辑。

- 今日概览、学习计划、目标管理、能力模型、图书详情、数据体检、成就管理的按钮层级统一到 `Button` 组件。
- 主操作使用默认 `Button`，同步、推送使用 `secondary`，导出和普通操作使用 `outline`，删除类操作使用 `destructive`。
- 图书详情入口文案从“AI阅读洞察”降级为“阅读洞察”，避免在 1.9.1 阶段强化 AI 预期。
- 数据体检描述从“用于 AI、报告和诊断之前”改为“用于后续报告和诊断之前”，保持 1.9.1 聚焦 UI 和数据质量边界。
- 修复学习计划页中断遗留的 JSX 标签不匹配问题。
