# 趣学伴 1.7.1 修复版本

更新时间：2026-04-30

## 版本定位

`1.7.1 = 数据固化 + 计划/设置/图书馆核心体验修复版`

1.7.1 不做 NotebookLM 接入，不做大规模 AI 能力扩展，不做服务器多应用迁移。  
本版本优先解决真实使用中的数据丢失、数据口径不统一、设置入口混乱、计划排序不稳定、图书馆阅读记录密度不足等问题。

## 一、数据固化与存储统一

### 目标

核心学习数据必须全部写入后端数据库。更换浏览器、清理缓存、发布新版前端后，数据不能丢失，也不能被旧本地缓存覆盖。

### 规则

- 核心业务数据严禁依赖 `localStorage` 作为唯一存储。
- 前端初始化时必须优先从 API 获取正式数据。
- `localStorage` 只允许保存 UI 偏好和短期交互状态。
- 本地缓存不得反向覆盖数据库中的正式业务数据。

### 必须迁移到数据库的数据

- 图书清单：当前 `library_book_lists_*`。
- 书籍详情笔记：当前 `book_notes_*`。
- 章节/目录配置：当前 `chapterStorageKey` 对应数据。
- 能力模型：`Family.settings.abilityModel` 或后续独立表。
- 目标管理：孩子目标、目标复盘、关联任务。
- 孩子档案：学期、阅读阶段、兴趣标签、推送配置。
- 任务记录：每日打卡、完成状态、延期、未完成、备注、证据。
- 阅读记录：阅读页数、分钟、日期、读后反馈、关联书籍/孩子。

### 允许保留本地的数据

- 主题、侧边栏折叠、页面筛选条件。
- 当前 tab、当前分页、搜索框临时输入。
- 当前选中孩子 ID。
- 临时弹窗状态和未提交表单草稿。

### 能力模型修复

必须排查“保存后刷新丢失”：

- 确认保存请求响应状态码和错误提示。
- 确认 `PUT /settings/ability-model` 写入 `Family.settings.abilityModel`。
- 确认 `GET /settings/ability-model` 返回数据库最新值。
- 确认页面初始化不会用默认模型覆盖已保存模型。
- 保存成功后刷新页面，应展示刚保存的能力模型。

## 二、数据质量预检闭环

当前数据质量预检只能发现问题，1.7.1 要补齐处理动作。

### 处理优先级

1. 缺页数。
2. 任务缺能力点。
3. 重复书。
4. 缺分类。
5. 缺 ISBN。

### 功能范围

- 批量补全页数。
- 批量设置图书分类。
- 重复书籍合并。
- ISBN / 封面自动补全。
- 任务与能力点批量关联。

### 验收标准

- 点击预检问题项后，能看到对应问题列表。
- 每类问题至少提供一个明确处理动作。
- 处理完成后重新预检，问题数量实时下降。
- 不允许只提示“发现问题”而不给处理入口。

## 三、计划发布与学习计划

### 任务排序规则

计划发布页任务展示顺序采用：

`时间段 > 任务状态 > 类别/学科 > 手动排序`

### 同步逻辑

- 同步计划后，保留用户已经手动调整好的排序。
- 不允许同步动作自动打乱已发布计划。
- 只有用户主动点击“一键重新编排”时，才按推荐规则重排。

### 时间精细度

采用“模糊时间段 + 精确提醒时间”：

- 页面展示使用时间段：早晨、放学后、晚饭后、睡前。
- Push 提醒使用精确时间：例如 18:30。
- 计划本体不强制精确到几点几分，避免孩子因几分钟误差产生挫败感。

### 状态图标统一

学习计划底部状态统一使用方形状态块：

- 完成：方形打钩。
- 进行中：进度条或播放图标。
- 未开始：空方块。
- 延期：时钟图标。
- 未完成：叉号图标。

### 字段补充

计划卡片增加：

- 计划次数。
- 已完成。
- 剩余。
- 今日状态。

## 四、孩子编辑与孩子概览

### 孩子编辑

- “阅读阶段”改为下拉框，范围：一年级、二年级、三年级、四年级、五年级、六年级、初一、初二、初三。
- “学习阶段”与“阅读阶段”联动，默认保持一致。
- 弱化或重命名“默认学习目标”，避免用户误解为正式目标管理。
- 弱化或重命名“能力偏好”，避免和能力模型混淆。

### 推送设置

每一项推送内容独立配置：

- 是否启用。
- 推送频率。
- 推送时间。

建议先支持：

- 任务提醒。
- 阅读提醒。
- 学习日报。
- 周报提醒。

### 学习数据

孩子编辑弹窗里的“学习数据”必须接入真实数据：

- 任务完成。
- 学习时长。
- 阅读页数/分钟。
- 周计划进度。

没有真实数据时显示空状态，不展示静态模拟图。

### 已修复入口与联动

- 新增孩子入口已补齐。
- 孩子概览已改为展示系统当前选中的孩子。

### 家长信息编辑

家长资料页补齐：

- 头像。
- 昵称。
- 修改密码。
- 登录信息。

### 家长管理权限

## 六、图书馆优化

### 阅读记录 UI 压缩

阅读记录改为单行极简：

`[封面小图] 书名 | 最近阅读时间 | 页数/分钟 | [状态方标]`

目标：

- 去掉大面积“书名 + 已读完”占位。
- 提高信息密度。
- 一屏内能看到更多阅读记录。

### 英文书管理

中英文书统一放在一个图书馆，不拆库。

规则：

- 新增 `语言` 字段。
- 默认语言为中文。
- 可选：中文、英文、双语、其他。
- 图书列表和统计页支持按语言筛选。
- 阅读统计可分语言查看，但总阅读量仍汇总在一起。

## 七、仪表盘阅读数据唯一事实源

### 统一口径

`reading_logs` 是阅读数据唯一事实源。

所有入口必须写入同一套阅读记录：

- 图书馆手动添加阅读记录。
- 每日任务中完成阅读类任务。
- 后续导入的阅读数据。

仪表盘只从 `reading_logs` 聚合：

- 阅读页数。
- 阅读分钟。
- 阅读天数。
- 最近阅读书籍。
- 阅读能力相关指标。

### 验收标准

- 图书馆记录一次阅读后，仪表盘同步更新。
- 每日任务完成阅读后，图书馆阅读记录同步出现。
- 不允许图书馆、任务、仪表盘各自维护一套阅读数据。

## 八、明确不纳入 1.7.1

- NotebookLM 接入。
- 正式 AI 自动决策。
- 服务器多应用目录迁移。
- 大规模重构任务/计划/能力模型数据结构，除非为数据固化必须。

- `pnpm --dir frontend build`

### 核心回归

- 登录。
- 当前孩子切换。
- 今日概览。

- 数据质量预检处理动作。

- 本地 UI 偏好不会覆盖数据库正式数据。
