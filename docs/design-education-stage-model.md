# 趣学伴教育阶段模型设计

更新时间：2026-05-01

## 一、设计结论

趣学伴后续采用“一套底层数据，双轨业务流”的阶段化设计。

核心原则：

- `educationStage` 是孩子画像的基础字段。
- 小学和初中使用不同的能力模型、目标口径、页面重点和报告框架。
- 能力模型是静态诊断地图，目标是动态执行计划，两者必须解耦。
- 1.8 先做阶段字段、能力标签和轻量页面适配。
- 2.0 做阶段化成长画像和报告。
- 3.0 再做阶段化 AI 学习伙伴。

## 二、核心字段

### educationStage

字段：

```ts
educationStage: 'primary' | 'middle'
```

含义：

- `primary`：小学阶段。
- `middle`：初中阶段。

存储位置：

- 用户/孩子画像表。
- 后续任务、打卡、阅读、表现记录可以在写入时保留阶段上下文，避免未来倒推历史数据。

默认策略：

- 新孩子默认可选择“小学阶段”。
- 老孩子升级后需要提示家长确认阶段。
- 不建议长期依赖本地缓存，必须写入后端数据库。

## 三、入口设计

入口：

```text
我的 / 孩子管理 / 编辑资料 / 学习阶段
```

交互：

- 使用下拉框或单选项。
- 选项：小学阶段、初中阶段。
- 修改后提示会影响今日概览、任务能力点、报告和钉钉推送口径。

切换提醒：

```text
切换学习阶段后，今日概览、能力模型和任务推荐将按新阶段展示。历史数据会保留，不会删除。
```

## 四、能力模型与目标解耦

### 能力模型

能力模型回答：

```text
孩子在哪些能力维度上发展？
```

它是相对稳定的诊断地图。

### 执行目标

执行目标回答：

```text
接下来具体做什么？
```

它是动态的行动计划，可以随周计划、任务、考试、阅读进度调整。

### 目标类型

建议新增目标类型：

```ts
targetType:
  | 'habit_process'
  | 'skill_growth'
  | 'knowledge_mastery'
  | 'exam_result'
  | 'reading_level'
```

含义：

- `habit_process`：习惯过程型。
- `skill_growth`：能力成长型。
- `knowledge_mastery`：知识掌握型。
- `exam_result`：考试结果型。
- `reading_level`：阅读等级型。

小学更多使用：

- `habit_process`
- `skill_growth`
- `reading_level`

初中更多使用：

- `knowledge_mastery`
- `exam_result`
- `skill_growth`

## 五、小学阶段 Primary Track

### 阶段定位

小学阶段更关注过程、习惯、素养和长期积累。

关键词：

- 习惯
- 专注
- 阅读
- 表达
- 基础能力
- 家长陪伴
- 稳定节律

### 能力维度

建议一级维度：

- 学习习惯
- 专注与执行
- 阅读理解
- 表达输出
- 基础语文
- 数学思维
- 英语核心素养
- 科学探索
- 运动健康
- 生活自理

说明：

- 不再使用“英语启蒙”作为正式维度，改为“英语核心素养”或在具体模块中使用“分级阅读与表达”。
- 小学高年级应预留分级阅读、数学思维、科学素养等数据，方便未来小升初能力差距分析。

### 执行目标特点

小学目标以过程型和成长型为主：

- 每天阅读是否持续。
- 是否能稳定完成晨读。
- 专注时长是否提升。
- 是否愿意表达和复述。
- 家长观察是否稳定记录。

### 时间调度

小学不应过早做精确到分钟的强排程。

建议支持时间块：

- 晨读
- 放学后
- 晚饭后
- 睡前
- 周末上午
- 周末下午

示例：

```text
晨读：完成 10 分钟朗读
睡前：亲子共读 15 分钟
周末上午：数学思维练习 20 分钟
```

### 今日概览重点

小学版今日概览优先展示：

- 今日任务完成情况。
- 阅读是否完成。
- 专注状态。
- 家长观察。
- 孩子反馈。
- 是否保持学习节律。

不建议过度强调排名、分数和考试压力。

## 六、初中阶段 Middle Track

### 阶段定位

初中阶段更关注结果、效率、学科掌握和复盘。

关键词：

- 学科
- 知识点
- 错题
- 效率
- 复习
- 考试
- 目标校
- 自主学习

### 能力维度

建议一级维度：

- 语文能力
- 数学能力
- 英语能力
- 科学/物理/化学
- 历史/地理/道法
- 错题复盘
- 思维逻辑
- 时间管理
- 考试策略
- 自主学习

说明：

- “思维逻辑”作为数理化底层能力维度，承接小学阶段的数学思维。
- 初中能力模型需要预留学科和知识点层级。

### 学科层级预留

初中建议预留两层结构：

```text
subject: 语文 / 数学 / 英语 / 物理 / 化学 / 生物 / 历史 / 地理 / 道法
abilityPoint: 阅读理解 / 古诗文 / 几何 / 函数 / 语法 / 完形 / 力学 / 错题复盘 ...
```

1.8 不必一次做完整知识图谱，但字段和配置结构要提前留好。

### 执行目标特点

初中目标以知识掌握和考试结果为主：

- 某知识点是否掌握。
- 错题是否复盘。
- 薄弱项是否减少。
- 学科投入是否有效。
- 阶段考试目标是否达成。

示例：

```text
数学：完成一次函数错题复盘
英语：完成阅读理解 2 篇并记录错因
物理：复习力学基础概念
```

### 时间调度

初中可以逐步支持更细的时间管理：

- 按学科分配时间。
- 番茄钟。
- 复习计划。
- 考前冲刺任务。
- 错题复盘任务。

### 今日概览重点

初中版今日概览优先展示：

- 学科投入时长。
- 完成任务数。
- 错题/薄弱点处理情况。
- 复习计划达成情况。
- 学习效率。
- 阶段目标进度。

## 七、报告与统计差异

### 小学报告

重点：

- 学习习惯稳定性。
- 阅读持续性。
- 专注力变化。
- 表达输出。
- 家长观察。
- 运动和生活习惯。

示例问题：

- 阅读是否连续？
- 专注是否比上周更稳定？
- 哪些任务需要家长陪伴？
- 孩子是否愿意表达？

### 初中报告

重点：

- 学科投入结构。
- 知识点掌握。
- 错题复盘。
- 薄弱项变化。
- 考试策略。
- 时间效率。

示例问题：

- 哪个学科投入多但效果弱？
- 哪些知识点反复出错？
- 错题是否被复盘和消灭？
- 当前表现和阶段目标之间差多少？

## 八、钉钉推送差异

### 小学阶段

文案重点：

- 鼓励家长关注陪伴、阅读、专注和习惯。

示例：

```text
今天的阅读和基础任务已完成，建议睡前用 5 分钟让孩子复述一个喜欢的内容。
```

### 初中阶段

文案重点：

- 关注学科、错题、复习和效率。

示例：

```text
今天数学投入较多，建议确认错题是否完成复盘，并标记仍未掌握的知识点。
```

## 九、AI 预留

1.8 不让 AI 直接介入决策，但必须保证数据结构可被 AI 使用。

AI 未来读取：

- `educationStage`
- `subject`
- `abilityCategory`
- `abilityPoint`
- `targetType`
- 任务记录
- 打卡结果
- 阅读记录
- 表现记录
- 家长观察

3.0 AI 应按阶段使用不同分析框架：

- 小学：习惯、阅读、专注、表达、陪伴。
- 初中：学科、知识点、错题、效率、目标差距。

## 十、与现有方案的区别

### 现有方案

当前系统更接近一套通用模型：

- 孩子没有明确教育阶段字段。
- 能力点更多是统一标签。
- 今日概览主要按任务和完成状态展示。
- 钉钉推送没有显著区分小学/初中。
- 报告和统计缺少阶段口径。

### 新方案

新方案升级为阶段化模型：

- 孩子画像新增 `educationStage`。
- 小学和初中使用不同能力维度。
- 能力模型和执行目标解耦。
- 小学偏过程和习惯。
- 初中偏结果和学科。
- 今日概览、报告、钉钉按阶段轻量分化。
- 数据结构为 2.0 成长画像和 3.0 AI 建议预留。

### 最大变化

过去：

```text
任务 -> 完成 -> 统计
```

未来：

```text
孩子阶段 -> 能力模型 -> 目标类型 -> 任务执行 -> 数据沉淀 -> 阶段化报告/AI
```

## 十一、版本落地

### 1.8：阶段化基础版

目标：先让数据按正确结构写入。

范围：

- 孩子编辑增加 `educationStage`。
- 能力点配置按小学/初中切换。
- 任务编辑保留阶段、学科、能力点、目标类型。
- 今日概览轻量阶段化。
- 钉钉文案轻量阶段化。
- 小学时间块任务做基础能力。
- 初中学科标签库做基础能力。

### 1.9：目标引擎升级版

目标：把能力诊断和目标执行真正解耦。

范围：

- 目标类型落地。
- 初中错题/薄弱项目标。
- 小学阅读/习惯目标。
- 周计划按阶段推荐不同目标。

### 2.0：阶段化成长画像版

目标：让家长看到阶段化成长趋势。

范围：

- 小学成长画像。
- 初中学科画像。
- 阶段化周报/月报。
- 小升初或阶段目标差距分析基础。

### 3.0：阶段化 AI 伙伴版

目标：AI 按阶段提供不同建议。

范围：

- 小学 AI 习惯与阅读建议。
- 初中 AI 学科和错题建议。
- 学习路径规划。
- 目标差距分析。

## 十二、1.8 优先级判断

问题：

```text
先做小学时间块调度，还是先做初中学科标签库？
```

建议：

```text
先做阶段字段和初中学科标签库，同时做小学时间块的最小版本。
```

原因：

- 阶段字段是所有后续数据的基础，必须第一优先级。
- 初中学科标签库决定后续任务、打卡、报告和 AI 能否沉淀正确数据。
- 小学时间块体验很重要，但第一版可以先做轻量化，不必一开始做复杂排程。

1.8 的实际优先顺序：

1. `educationStage` 字段和孩子编辑入口。
2. 小学/初中能力配置表。
3. 初中学科标签库基础版。
4. 任务编辑页按阶段切换能力点。
5. 小学时间块调度最小版。
6. 今日概览和钉钉轻量阶段化。

## 十三、暂不做

1.8 暂不做：

- 完整知识图谱。
- 目标学校数据库。
- 名校差距量化看板。
- AI 学习路径规划。
- 大规模页面重构。

这些放到 2.0 或 3.0。
