# 生产环境收口计划

## 当前结论

- 线上环境、本地工作区、GitHub `origin/main` 一度不一致，后续修复以 `release/fix-core-flows` 为唯一修复线。
- 服务器上曾经存在手改前后端代码、手动启动后端进程、PM2 失效、nginx 静态目录与前端构建目录分离的问题。
- 生产数据库 migration 历史已漂移，本轮不处理 schema 治理，只围绕核心功能恢复可用。

## 已确认的服务器热修

- 设置页裁剪为只保留“孩子管理”，避免 `AccountSettings` 等模块在生产环境运行时报错。
- 前端静态资源已通过 `frontend/dist -> /var/www/study-planner` 的方式发布。
- 线上后端实际提供的登录/注册/添加孩子接口仍然是：
  - `POST /api/login`
  - `POST /api/register`
  - `POST /api/add-child`

## 当前核心目标

只恢复这三条链路：

1. 总览看板
2. 任务管理
3. 计划发布

## 修复优先级

### P0

- 登录链路前后端路径统一
- 孩子管理可用
- 任务列表 / 新建 / 编辑 / 删除可用
- 计划发布可用

### P1

- 看板数据与当前孩子保持一致
- 后端进程交给 PM2 正式托管
- 前端部署与 nginx 指向目录固化

### P2

- 恢复完整设置页
- 图书馆与其他非核心模块
- 数据库 migration 历史治理

## 任务管理当前已知问题

- 旧任务虽然已补绑到当前孩子，但编辑/删除仍然返回 500，需要抓正在运行的后端进程日志。
- 创建任务返回 500，需要后端日志确认。
- 必须先统一“当前在线后端进程”与正式部署方式，再继续查任务相关 500。

## 接下来的执行顺序

1. 确认并收口当前在线后端进程的启动方式
2. 抓任务新建 / 编辑 / 删除的真实后端报错
3. 修复任务 CRUD
4. 验证计划发布
5. 回归看板

## 暂不处理

- 图书馆相关问题
- 报告 / 成就 / 统计模块
- Prisma migration 漂移
- 服务器历史残留脚本与无关文件
