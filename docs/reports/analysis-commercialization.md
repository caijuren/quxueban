## 1️⃣ 系统功能概述

- 周计划制定
- 每日打卡
- 进度追踪
- 灵活配置（跳过假期等）

#### 📚 阅读管理
- 书籍图书馆
- 书籍信息管理（作者、ISBN、页数、字数等）
- 阅读进度记录
- 阅读日志

#### 🤖 AI功能
- 书籍智能分析
- 学习洞察报告
- AI配置管理

#### 📊 数据与统计
- 学习统计与趋势
- 成就系统
- 报表生成
- 数据导出

#### 🔔 集成功能
- 钉钉机器人通知

## 2️⃣ 技术架构分析

### 2.1 技术栈

#### 后端
- **框架**：Express.js 4.x
- **语言**：TypeScript
- **数据库**：PostgreSQL + Prisma ORM
- **认证**：JWT
- **日志**：Pino
- **验证**：Zod
- **安全**：CORS、输入验证、错误处理

- **框架**：React 19
- **构建工具**：Vite
- **UI组件**：Radix UI + Tailwind CSS
- **状态管理**：React Context API
- **HTTP客户端**：Axios
- **路由**：React Router
- **图表**：Recharts

## 3️⃣ 核心业务逻辑

### 3.1 认证流程

```typescript
// 用户登录
POST /api/login
{
  username: string,
  password: string
}
// 返回 JWT token + 用户信息
```

**关键安全点**：
- JWT 签名密钥（默认 `dev-secret-key`）⚠️
- Token 过期时间（默认7天）
- 仅允许 `parent` 角色登录
- 密码用 bcrypt（12轮）加密

### 3.2 任务创建与分配

1. 家长创建任务
2. 选择适用的孩子（`appliesTo` 字段）
3. 配置任务属性（分类、类型、时间、难度等）
4. 存储到数据库（带 `familyId` 隔离）

### 3.3 周计划发布

1. 选择任务集合
2. 配置周期
3. 分配给孩子
4. 生成周计划记录

### 3.4 数据隔离逻辑

```typescript
// 从 JWT 中提取 user
{
  userId: number,
  role: string,
  familyId: number,
  name: string
}

// 查询时必须加上 familyId 过滤
where: { familyId: req.user.familyId }
```

## 4️⃣ 发现的问题与安全漏洞 🔴

### 4.1 严重安全问题 🔴

#### 🔴 问题1：JWT密钥不安全
**位置**：`backend/src/config/env.ts`
```typescript
JWT_SECRET: z.string().default('dev-secret-key'),
```

**问题描述**：
- 默认密钥硬编码，生产环境未强制变更
- 密钥泄露后可伪造任意用户的token
- 可能导致未授权访问和数据泄露

**风险级别**：严重

**修复建议**：
- 移除默认值，生产环境必须配置
- 密钥长度至少32个字符
- 使用环境变量管理，不提交到代码库

---

#### 🔴 问题2：数据隔离验证不完整 ⚠️
**位置**：多个业务模块

**问题描述**：
- 部分查询仅依赖 `familyId` 过滤，但缺少验证
- 用户可能尝试访问其他家庭的数据（通过猜测ID）
- 例如：删除任务、查看其他家庭的书籍等

**相关文件**：
- `backend/src/modules/tasks.ts`
- `backend/src/modules/library.ts`
- `backend/src/modules/plans.ts`
- 等等...

**风险级别**：高

**修复建议**：
- 所有写操作和敏感读操作必须验证资源归属
- 通用验证中间件：
```typescript
export const verifyOwnership = async (
  resourceType: string,
  resourceId: number,
  familyId: number
) => {
  const resource = await prisma[resourceType].findUnique({
    where: { id: resourceId }
  });
  
  if (!resource || resource.familyId !== familyId) {
    throw new AppError(403, 'Access denied');
  }
  return resource;
};
```

---

#### 🔴 问题3：缺少速率限制
**位置**：认证接口和敏感操作接口

**问题描述**：
- 登录接口无速率限制
- 暴力破解密码风险
- API可能被滥用

**风险级别**：高

**修复建议**：
```typescript
import rateLimit from 'express-rate-limit';

const loginLimiter = rateLimit({
  windowMs: 15 * 60 * 1000, // 15分钟
  max: 5, // 最多5次
  message: 'Too many login attempts'
});

authRouter.post('/login', loginLimiter, ...);
```

---

#### 🔴 问题4：CORS配置过于宽松
**位置**：`backend/src/app.ts`
```typescript
cors({
  origin: env.CORS_ORIGIN.includes(',')
    ? env.CORS_ORIGIN.split(',').map(s => s.trim())
    : env.CORS_ORIGIN,
  credentials: true,
})
```

**环境配置**：
```typescript
CORS_ORIGIN: z.string().default('http://localhost:5173,http://localhost:8080'),
```

**问题描述**：
- 本地开发默认允许两个来源
- 生产环境可能错误使用 wildcard
- Credentials 设为 true，若 origin 为 * 会有安全问题

**风险级别**：中

---

### 4.2 功能与数据问题 ⚠️

#### ⚠️ 问题5：密码复杂度要求低
**位置**：`backend/src/modules/auth.ts`
```typescript
if (password.length < 6) {
  throw new AppError(400, '密码至少6位');
}
```

**问题描述**：
- 仅检查长度≥6位
- 无复杂度要求（大小写、数字、特殊字符）
- 弱密码容易被破解

---

#### ⚠️ 问题6：用户枚举漏洞
**位置**：登录和注册接口

**问题描述**：
- 注册时，用户名已存在返回409
- 登录时，用户不存在和密码错误都返回401（消息相同）✅ 这个做得好！
- 但注册接口仍存在枚举风险

---

#### ⚠️ 问题7：任务系统设计混乱

**问题描述**：
- 有两个任务系统：`Task` 和 `TaskTemplate` + `ChildTask`
- 功能重叠，逻辑混乱
- `weekly_plans` 关联的是 `Task` 表
- 但 `Task` 表几乎被废弃，使用的是 `ChildTask` 方式
- 备份数据显示有 `Task` 数据，但新代码可能走 `TaskTemplate` 路径

**影响**：
- 数据恢复困难
- 功能迁移风险
- 维护成本高

---

#### ⚠️ 问题8：缺少日志审计
**问题描述**：
- 敏感操作（删除任务、修改设置、导出数据）无审计日志
- 无法追溯用户操作
- 出现问题难以调查

---

#### ⚠️ 问题9：上传文件安全风险
**位置**：`backend/src/modules/upload.ts`（可能）

**问题描述**：
- 文件上传未验证文件类型
- 可能被上传恶意文件
- 文件存储路径不安全
- 文件名处理可能有路径遍历漏洞

---

#### ⚠️ 问题10：数据库连接信息
**位置**：环境配置

**问题描述**：
- 使用单个数据库用户，权限过大
- 未使用数据库连接池优化
- 缺少数据库备份策略（虽然有备份脚本）

---

## 5️⃣ 改进建议 ✅

### 5.1 安全性改进（优先级：P0）

#### P0 - 必须修复
1. **更换 JWT 密钥**：使用强随机密钥，环境变量管理
2. **完善数据隔离验证**：所有操作验证 `familyId` 一致性
3. **添加速率限制**：认证接口、敏感操作接口
4. **安全的 CORS 配置**：生产环境使用明确的允许列表
5. **文件上传安全加固**：类型验证、病毒扫描、文件名安全处理

#### P1 - 应该修复
6. **密码策略升级**：复杂度要求、密码过期、历史密码检查
7. **审计日志**：记录所有敏感操作
8. **会话管理**：Token黑名单、强制登出
9. **SQL注入防护**：确认Prisma使用参数化查询（应该没问题，但要确认）

#### P2 - 建议优化
10. **CSRF 保护**：虽然用JWT在header中，但可以添加Double Submit Cookie
11. **安全头**：添加 X-Content-Type-Options, X-Frame-Options, CSP 等
12. **依赖更新**：定期更新 npm 包，修复安全漏洞

---

### 5.2 功能改进（优先级：P1）

1. **统一任务系统**：明确是用 Task 还是 TaskTemplate + ChildTask，重构代码
2. **完善的测试**：单元测试、集成测试覆盖率≥80%
3. **数据验证**：前后端都添加完整的验证逻辑
4. **错误处理标准化**：统一错误码、错误信息格式
5. **前端状态管理优化**：考虑使用 Zustand 或 Redux Toolkit

---

### 5.3 运维改进（优先级：P1）

1. **容器化部署**：Docker + Docker Compose
2. **CI/CD**：自动化构建、测试、部署
3. **备份策略**：定时自动备份，异地存储
4. **监控告警**：Prometheus + Grafana 或其他方案
5. **日志聚合**：ELK 或类似方案

---

### 5.4 用户体验改进（优先级：P2）

1. **加载状态**：所有 API 调用都有 Loading 状态
2. **错误重试**：网络错误自动重试机制
3. **离线体验**：考虑 PWA 支持
4. **键盘快捷键**：提升操作效率
5. **多语言支持**：国际化框架
