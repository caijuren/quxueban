# 趣学伴整体 ToDo

更新时间：2026-05-01

## 当前执行原则

本文件是当前唯一的总 ToDo。旧版本任务、历史发布记录和旧路线图已经按类型归档到 `docs/todos/`、`docs/releases/` 和 `docs/roadmaps/`。

当前产品方向：

- 小学 1.9 正式采用“三层准备度模型”：交付层、认知层、稳定性层。
- 小学三公方向不再按“五大核心 + 四个支撑模型”同级展示；五大核心归入交付层，执行支撑拆入稳定性层、数据证据和诊断引擎。
- 初中版本采用同一底层引擎，但换成初中目标配置：学科交付、认知方法、稳定性系统。
- 1.8 只收尾当前已经展开的任务详情和阶段化工作。
- 1.9 的重点是三层模型、目标引擎、任务模型整理、稳定性层基础数据和轻量诊断卡片。
- 能力掌握度、复杂诊断、目标校适配度、仪表盘重构和 AI 报告都等 1.9 基础闭环结束后再做。
- 2.0 前完成主体功能闭环；2.0 的核心是全面引入 AI，让系统真正被 AI 驱动。

## P1：1.9 三层准备度模型与目标引擎

设计入口：`docs/design-1.9-ability-goal-engine.md`。

### 5. 三层准备度模型定型

状态：1.9.0 首轮已落地，后端配置化和权重体系进入 1.9.x。

范围：

- 小学三公模板定型为交付层、认知层、稳定性层。
- 交付层包含英语、数理、大语文、项目成果、临场应变。
- 认知层包含规则内化、迁移应用、信息处理速度、逻辑深度、批判思维。
- 稳定性层包含睡眠、运动、情绪、完成率、延期率、复盘频率和外部负载。
- 初中版本按同一底层抽象预留：学段、目标、层级、模块、指标、规则、权重、建议模板。
- 家庭级编辑先轻量化，优先允许编辑目标和能力点文案；全层级模板编辑后置。

### 6. 目标引擎升级

状态：1.9.0 首轮已落地。

范围：

- 目标体系与三层准备度模型匹配。
- 目标成为独立对象，绑定能力点、周期、状态、成功标准和复盘节奏。
- 小学目标支持三公基础模板，后续扩展上实、上外附中、浦外权重。
- 初中目标支持中考、自招、特色高中、国际路线等后续目标配置。
- 任务必须能解释服务哪个目标和哪个能力点。
- 目标完成情况进入首页建议、目标页和报告摘要。

### 6.1 轻量诊断卡片

状态：1.9.0 首轮已落地。

范围：

- 首页或能力模型页增加“本周建议”区域。
- 第一版只输出冲刺、修复、减压、维持四类策略。
- 首批诊断覆盖高压透支、执行断裂、外部扰动、数据不足。
- 所有建议必须显示证据来源或“证据不足”提示。
- 复杂诊断，例如假性退步、假性爆发、认知阻力位，放入 1.9.x。

### 6.2 稳定性层基础数据

状态：1.9.0 首轮已落地，独立 schema 和统计 API 后置。

范围：

- 完成率、延期率、复盘频率进入稳定性层。
- 睡眠、运动、情绪进入稳定性层。
- 增加外部负载记录：考试周、生病、旅行、换老师、家庭节奏异常。
- 稳定性层支持高压透支和执行断裂的基础判断。

### 6.3 数理认知试点字段

状态：1.9.0 首轮已落地，当前只沉淀证据，不做自动评分。

范围：

- 数理任务支持认知标签：规则内化、迁移应用、信息处理速度、逻辑深度。
- 记录尝试次数、耗时、是否提示、错因、复盘质量。
- 1.9 只沉淀证据，不做自动认知评分。

### 6.4 任务列表分配字段重构

状态：1.9.0 首轮已落地。

范围：

- 重新梳理任务列表里的“分配规则、频次、记录方式、阶段字段、目标类型、时间块”的展示优先级。
- 区分交付层、认知层、稳定性层任务的列表摘要，不再把旧字段机械堆叠在卡片上。
- 对旧任务缺字段给出低干扰提示，避免列表页显示不合理标签。

### 7. 学制配置

状态：未开始，可并入 1.9 或 1.9.x。

范围：

- 孩子档案增加 `schoolSystem`：五四制、六三制。
- 阶段判断支持 `schoolSystem + grade -> educationStage`。
- 五四制：1-5 年级小学，6-9 年级初中。
- 六三制：1-6 年级小学，7-9 年级初中。
- 小学三层准备度模型可按低段、中段、高段表达，高段对应小学最后 1-2 年。

不做：

- 完整课程进度。
- 精细知识点体系。

### 8. 数据质量轻量版

状态：1.9.x 首版已落地，后续根据真实缺口补批量修复能力。

解释：

数据质量轻量版是“数据体检入口”，用于发现 AI、能力掌握度和报告之前的数据缺口。它不是复杂中心，第一版只做检查和跳转修复。

候选检查范围：

- 任务缺能力点。
- 任务缺目标类型。
- 目标缺能力点。
- 图书缺 ISBN、页数、类型。
- 阅读记录缺日期、页数、时长。
- 打卡记录缺质量评价或家长观察。

首版已完成：

- 新增 `/parent/data-quality` 数据体检入口。
- 复盘分析导航增加“数据体检”。
- 按当前孩子聚合任务、目标、图书、在读记录和指定日期打卡数据。
- 展示缺口数量、比例、示例和跳转修复入口。
- 检查记录见 `docs/checks/data-quality-1.9x.md`。

V0.1 已完成：

- 当前趋势基于近 8 周打卡，比较最近 4 周和前 4 周完成率，并补充近 14 天变化。
- 已增加余力指数首版，综合 Velocity、负向状态、稳定性波动、负载信号和证据覆盖率。
- 已增加跨学科交叉验证首版，按英语、数理、大语文对比完成率和负向状态，区分单科问题和通用问题。
