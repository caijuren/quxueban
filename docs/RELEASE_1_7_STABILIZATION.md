# 趣学伴 1.7 稳定期计划

更新时间：2026-04-29

## 版本定位

`1.7.x = 1.7 功能稳定、生产修复、进入 1.8 前的冻结期`

1.7 已经完成图书馆导入预检、数据质量摘要、任务能力点、孩子切换、头像和生产部署脚本修复。接下来先稳定使用一段时间，只处理真实使用中暴露的问题。

## 版本节奏

- `1.7.0`：当前线上功能版。
- `1.7.1`：数据固化、数据质量闭环、计划排序、设置/孩子/图书馆核心体验修复版，详见 `docs/RELEASE_1_7_1.md`。
- `1.7.2 - 1.7.4`：小步修复版，只处理 bug、核心链路体验、部署稳定性和低风险文案调整。
- `1.7.5`：1.7 稳定汇总版，作为启动 1.8 前的冻结基线。
- `1.8.0`：在 1.7.5 稳定后启动，进入数据闭环、能力模型增强和多应用服务器治理。

## 1.7.x 允许做

- 修复登录、今日概览、任务管理、计划、图书馆、设置等核心链路 bug。
- 修复生产部署脚本、PM2、nginx、版本号、静态资源发布等上线问题。
- 修复移动端明显遮挡、按钮不可点、弹窗不可滚动等使用问题。
- 补充缺失的空状态、错误提示、加载状态。
- 修正明显错误的统计口径或数据过滤。
- 补小范围测试，覆盖已复现的问题。
- 每次发布同步更新 package 版本和 `/api/version`。

## 1.7.x 不做

- 不迁移 quxueban 服务器目录。
- 不引入多应用服务器结构调整。
- 不新增大型页面或复杂模块。
- 不重构任务、计划、能力模型的大结构。
- 不做自动部署、tag 部署或 CI/CD 大改。
- 不引入正式 AI 决策流程。
- 不做大规模数据库模型调整，除非是生产问题必须修。

## 每次 1.7.x 发布检查

1. 本地工作区只包含本次修复相关文件。
2. `backend/package.json`、`frontend/package.json`、根 `package.json` 和 `backend/src/modules/system.ts` 版本号一致。
3. 后端构建通过：`pnpm --dir backend build`。
4. 前端构建通过：`pnpm --dir frontend build`。
5. 如果涉及数据库，先本地验证迁移，再生产执行 `prisma migrate deploy`。
6. 生产部署使用 `bash scripts/deploy-production.sh`，不要手动删除 `node_modules`。
7. 部署后验证：
   - `/api/health`
   - `/api/version`
   - PM2 `study-planner-api` 在线
   - nginx 静态资源已更新
   - 登录
   - 今日概览
   - 图书馆
   - 任务管理
   - 计划

## 1.7.5 冻结标准

- 连续使用一段时间没有 P0/P1 生产问题。
- 核心链路能稳定完成：登录、孩子切换、今日打卡、任务编辑、计划查看、图书导入、图书状态更新。
- 生产部署脚本至少成功跑过一次完整流程。
- `/api/version` 与 package 版本一致。
- PM2 进程表稳定，重启后能恢复。
- nginx 静态资源发布路径明确。
- 1.8 待办清单已经整理，不混入 1.7.x 修复分支。

## 进入 1.8 前置条件

- 1.7.5 已发布并稳定。
- 1.8 范围冻结：数据闭环、能力模型、任务编辑统一、多应用服务器治理等明确优先级。
- 服务器迁移只在 1.8 维护窗口执行，迁移前保留完整回滚方案。
- 1.8 开始后不再把 1.7.x 修复和 1.8 结构性改动混在同一提交里。
