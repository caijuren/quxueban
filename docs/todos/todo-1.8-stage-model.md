- 已完成 P0-3：新增阶段化能力配置，任务表单按小学/初中切换能力点。

## 1.8 版本目标

1.8 只做“阶段化基础版”，不做大规模页面重构。

目标一句话：

```text
让系统知道孩子处于小学还是初中，并让任务、能力点、今日概览和钉钉开始按阶段写入和展示。
```

## 1.8 必做范围

### P0-1：教育阶段字段

任务：

- 后端孩子模型增加 `educationStage`。
- 可选值：`primary`、`middle`。
- 默认值：`primary`。
- 孩子创建、编辑、列表、详情都返回该字段。
- 兼容旧孩子数据。

验收：

- 老孩子不崩溃，默认按小学阶段展示。
- 新建孩子可以选择小学/初中。
- 编辑孩子可以修改学习阶段。

### P0-2：孩子编辑入口

任务：

- 在孩子编辑页增加“学习阶段”选择。
- 选项：小学阶段、初中阶段。
- 修改时提示：阶段会影响能力点、任务推荐、今日概览和钉钉推送。

验收：

- 保存后刷新页面仍保留阶段。
- 切换孩子时阶段不串。

### P0-3：阶段化能力配置

任务：

- 新增前端能力配置文件。
- 小学能力维度：
  - 学习习惯
  - 专注与执行
  - 阅读理解
  - 表达输出
  - 基础语文
  - 数学思维
  - 英语核心素养
  - 科学探索
  - 运动健康
  - 生活自理
- 初中能力维度：
  - 语文能力
  - 数学能力
  - 英语能力
  - 科学/物理/化学
  - 历史/地理/道法
  - 错题复盘
  - 思维逻辑
  - 时间管理
  - 考试策略
  - 自主学习

验收：

- 任务表单能根据当前孩子阶段展示对应能力维度。
- 不再把小学和初中能力点混在同一个选择列表里。

### P0-4：目标类型

任务：

- 任务模型或任务标签中增加 `targetType`。
- 可选值：
  - `habit_process`
  - `skill_growth`
  - `knowledge_mastery`
  - `exam_result`
  - `reading_level`
- 任务表单增加目标类型选择。

验收：

- 小学任务默认推荐过程型、成长型、阅读等级型。
- 初中任务默认推荐知识掌握型、考试结果型、能力成长型。
- 旧任务没有目标类型时不影响展示。

### P0-5：初中学科标签库基础版

任务：

- 新增初中学科选项：
  - 语文
  - 数学
  - 英语
  - 物理
  - 化学
  - 生物
  - 历史
  - 地理
  - 道法
- 任务表单在初中阶段展示学科选择。
- 先不做完整知识图谱。

验收：

- 初中孩子创建任务时可以选择学科。
- 学科信息保存到任务数据中。

### P0-6：小学时间块最小版

任务：

- 新增时间块选项：
  - 晨读
  - 放学后
  - 晚饭后
  - 睡前
  - 周末上午
  - 周末下午
- 小学阶段任务表单可选择时间块。
- 初中阶段暂不强依赖时间块。

验收：

- 小学任务可以保存时间块。
- 今日概览或任务列表能展示时间块。

### P1-1：今日概览轻量阶段化

任务：

- 小学阶段文案强调习惯、阅读、专注、家长观察。
- 初中阶段文案强调学科、错题、复习、效率。
- 不做双页面布局。

验收：

- 当前孩子是小学时，建议文案不出现初中应试导向。
- 当前孩子是初中时，建议文案不再只强调低龄习惯。

### P1-2：钉钉轻量阶段化

任务：

- 小学推送文案强调阅读、专注、习惯。
- 初中推送文案强调学科、错题、复习。

验收：

- 同一天数据下，小学/初中孩子推送文案重点不同。
- 学习时长、完成率等数字口径不变。
