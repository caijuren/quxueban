# 趣学伴 1.5 遗留问题清单

更新时间：2026-04-27

## 分级说明

- `P0`：正式上线前必须处理或确认降级，否则影响核心使用。
- `P1`：可以上线，但需要在 1.5.x 或 1.8 前处理。
- `P2`：体验、统一性、长期维护问题，可进入后续版本。

## P0 上线前必须确认

| 编号 | 问题 | 当前判断 | 建议处理 |
| --- | --- | --- | --- |
| 1.5-P0-01 | 核心页面是否存在崩溃/白屏/弹窗无法关闭 | 自动化通过，仍需最后浏览器冒烟 | 重点看今日概览、任务、计划、图书馆、仪表盘 |
| 1.5-P0-02 | 孩子切换是否影响核心页面数据刷新 | 接口和上下文已可用 | 浏览器冒烟确认顶部切换入口 |
| 1.5-P0-03 | 数据导出 500 | 已修复：导出文件名改用家庭编码，避免中文 header | 当前代码复测 200；上线后再测生产环境 |

| --- | --- | --- | --- |

## P2 体验与工程债

| 编号 | 问题 | 当前判断 | 建议处理 |
| --- | --- | --- | --- |
| 1.5-P2-01 | 全站仍有部分弹窗和页面风格不完全统一 | 需要 UI 系统化巡检 | 建立 UI 组件规范清单 |
| 1.5-P2-02 | Vite 构建存在大 chunk 提示 | 不阻塞上线，但影响长期性能 | 后续做路由级懒加载和手动分包 |
| 1.5-P2-03 | 自动化测试覆盖不足 | 目前测试量偏少，主要依赖人工回归 | 补充任务、计划、图书、设置关键用例 |
| 1.5-P2-04 | 生产 migration 历史不完全一致 | 发布时需谨慎 | 单独做数据库迁移治理 |
| 1.5-P2-05 | 页面数据源分散，口径不统一 | 会影响 1.8 和 2.0 的可信度 | 建立数据字典和指标口径 |
| 1.5-P2-06 | 后端 mock mode 代码仍较多 | 正常数据库模式不使用，但容易干扰代码理解 | 1.8 工程治理时清理或隔离 |

## 建议封板策略

1. `P0` 必须在正式上线前逐项确认。
2. 如果某个 `P0` 暂时不修，必须从入口隐藏或降级为不可点击说明，不能保留崩溃入口。
3. `P1` 全部进入 1.8/2.0 规划，不阻塞 1.5 首次上线。
4. `P2` 进入工程质量和体验债务池。
