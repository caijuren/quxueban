# 趣学伴 1.5 封板与上线说明

更新时间：2026-04-27

## 版本定位

`1.5 = 家长端体验统一与上线收口版`

本版本的重点不是新增大模块，而是把家长端主流程整理成可以正式上线使用的状态：

- 主导航收口为家长端管理体系
- 页面顶部操作区统一
- 时间选择器统一
- 今日概览、任务、计划、图书馆、设置等核心页面完成关键可用性修正
- 移除明显的孩子端残留、静态演示感和无效入口
- 为 1.8 的管理体系升级和 2.0 的 AI 升级预留数据与产品方向

## 1.5 已完成范围

### 1. 导航与信息架构

- 主导航从“成长仪表盘”调整为“仪表盘”。
- 孩子端相关页面不再作为当前主线入口。
- 家长端主流程聚焦为：仪表盘、目标、今日概览、任务管理、学习计划、图书馆、学习报告、成就、设置。
- 孩子切换从侧边栏移到顶部通知按钮左侧，降低全局上下文切换成本。

### 2. 全站顶部操作区统一

- 新增并使用公共顶部长条组件 `PageToolbar`。
- 今日概览、仪表盘、目标管理使用统一粉紫渐变长条。
- 任务管理顶部控制条同步使用同款外观。
- 相关页面容器已统一到今日概览标准宽度 `max-w-[1360px]`。
- 同类按钮配色向学习计划页靠拢：蓝色用于同步/通知类，绿色用于导出类，紫色渐变用于主要创建/调整类操作。

### 3. 时间选择体验统一

- 新增公共 `DatePicker` 组件。
- 已替换前端原生 `type="date"` 输入。
- 今日概览、仪表盘自定义时间范围、任务/计划相关日期选择统一为弹出式日历交互。
- 仪表盘保留“自定义”入口，并支持选择时间段。

### 4. 今日概览

- 顶部操作区保留日期选择、安排计划、钉钉、导出。
- 移除顶部重复 AI 分析入口，AI 内容保留在页面主体中。
- 今日任务、学习趋势、AI 建议 + 快捷操作三列改为同高、底部对齐。
- 今日任务区域改为内部滚动，避免整页被任务列表拉长。
- 安排计划按钮颜色已与钉钉按钮区分。

### 5. 仪表盘

- 删除原“统计范围”文字和起止日期裸露展示。
- 阅读能力、自定义等按钮重新排布，避免折叠异常。
- 自定义时间段使用与今日概览一致的时间选择模式。
- 仪表盘顶部与其他页面统一为同款长条。

### 6. 目标管理

- 新增目标管理页面入口。
- 第一行按钮配色已统一。
- 页面顶部长条与今日概览同宽、同内部节奏：图标块、标题、描述、右侧操作。
- 当前目标内容仍偏静态，作为 1.8 管理体系升级的基础页面。

### 7. 任务管理

- 删除卡片/列表切换，默认保留卡片模式。
- 删除原页面第一行说明区。
- 原第一行操作按钮与筛选按钮整合进任务池结构下方的统一长条。
- 任务详情进入编辑任务时的页面风格已向整体设计靠拢。
- 运营健康度暂保留当前展示方式，后续单独处理。

### 8. 学习计划

- 日期选择与计划调整体验继续保留并作为全站时间选择参考。
- 每个日期下的任务方块增加间距，避免多个任务色块贴在一起。
- 计划更新接口已修复按 `childId/familyId` 定位计划的问题，降低刷新后丢失风险。

### 9. 图书馆与图书详情

- 图书馆旧入口已收口：批量操作、年龄筛选、快捷编辑/借出等低频入口不作为 1.5 主流程。
- 图书卡片补充删除入口。
- 图书封面显示和头像/图片显示方式向等比裁切靠拢，避免直接拉伸。
- Excel/CSV 导入仍需要上线前专项复测。

### 10. 设置与家庭管理

- 设置页快速导航中明显无效入口已修正或收口。
- 假通知、假的操作历史等静态演示感内容已移除。
- 头像展示改为等比裁切，不再直接压缩变形。
- 孩子管理、家庭成员、通知配置、危险操作仍作为 1.5 遗留重点记录。

### 11. 后端与数据能力

- 后端任务、计划、阅读、图书馆等模块已进行多处稳定性修正。
- 学习计划更新、阅读进度同步、图书状态等关键链路已补强。
- 当前 1.5 更偏“可上线体验收口”，不是完整数据模型重构版本。

## 上线前必须完成的验证

### 基础命令

前端：

```bash
cd frontend
pnpm lint
pnpm build
pnpm test -- --run
```

后端：

```bash
cd backend
pnpm lint
pnpm build
pnpm test --runInBand
```

数据库与服务：

```bash
cd backend
npx prisma migrate status
curl http://localhost:3001/api/health
```

注意：生产库 migration 历史目前与源码 migration 目录不完全一致，上线前不要执行 `prisma migrate reset`。

### 人工回归清单

1. 登录、退出、刷新保持登录态。
2. 孩子切换后，今日概览、任务、计划、图书馆数据刷新。
3. 今日概览日期切换、任务打卡、钉钉、导出入口。
4. 任务新建、编辑、删除、筛选、进入详情再编辑。
5. 学习计划周切换、日期选择、任务分配、保存后刷新保持。
6. 图书馆新增、编辑、删除、搜索、筛选、封面显示。
7. 图书详情开始阅读、更新进度、完成阅读、阅读记录同步。
8. Excel/CSV 导入是否成功，是否有明确结果反馈。
9. 设置页：修改头像、修改用户名、修改密码、孩子管理、家庭设置、通知配置。
10. 危险操作是否有二次确认，不允许误触。
11. 全站弹窗、顶部长条、按钮颜色、时间选择器是否统一。
12. 手机宽度下顶部孩子切换、通知、侧边栏是否可用。

## 线上发布步骤

以前端 nginx 根目录 `/var/www/study-planner`、后端 PM2 进程 `study-planner-api` 为准。

### 前端

```bash
cd /home/ubuntu/frontend
git pull origin main
pnpm build
sudo rsync -av --delete /home/ubuntu/frontend/dist/ /var/www/study-planner/
sudo systemctl restart nginx
```

### 后端

```bash
cd /home/ubuntu/backend
git pull origin main
pnpm install
pnpm build
pm2 restart study-planner-api
pm2 status
curl http://localhost:3001/api/health
```

## 上线后观察

- 登录失败率、接口 401/500。
- 今日概览、学习计划、图书馆接口响应时间。
- 孩子切换后数据是否串号。
- 图书删除、任务删除、危险操作是否误触。
- 钉钉、导出、导入类功能的错误提示是否明确。
- 移动端顶部操作区是否遮挡。

## 1.5 封板结论

1.5 可以进入封板准备，但建议在正式发布前完成一次完整人工回归，并把 `docs/ISSUES_1_5.md` 中的 P0 项清零或明确降级。
