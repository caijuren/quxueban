# 趣学伴 1.7.3 发布计划

更新时间：2026-04-30

## 版本定位

`1.7.3 = 任务/计划体验精修 + 孩子编辑表单修复 + 上线稳定版`

1.7.3 是 1.7 稳定期内的小步上线版本，不做 1.8 的能力快照、自动算分、事件日志和数据库大结构调整。  
本版本重点把家长每天高频使用的 `任务管理`、`学习计划`、`今日概览联动`、`孩子编辑` 打磨到更适合真实使用。

## 一、已纳入修复

### 孩子编辑

- 修复 `孩子编辑 > 学期与学习` 中下拉菜单、日期选择器和文本框大小不一致的问题。
- 统一字段网格、标签、控件高度、圆角和宽度。
- 保持“学期与学习”仍是配置入口，不把它误改成正式能力模型页。

验收：

- 学年、学期、年级、开始日期、结束日期、阅读阶段排列整齐。
- 学习阶段、默认能力级别、阶段备注排列整齐。
- 前端构建通过。

## 二、任务管理改造

### 目标

任务页不再继续堆字段，而是让家长更清楚：

- 这个任务是什么。
- 适合什么阶段。
- 训练哪个能力点。
- 当前是否缺关键配置。
- 是否已经能顺利进入计划和今日概览。

### 1. 任务卡片信息层级

保留卡片主信息：

- 任务名称。
- 类型/学科/频率。
- 单次时长。
- 能力点。
- 关联目标。

弱化或折叠低频信息，避免卡片过重。

### 2. 配置完整性提示

任务列表增加轻量状态提示：

- 缺能力点。
- 缺目标关联。
- 缺记录方式。
- 缺计划规则。

提示只做提醒，不阻断创建和编辑，避免影响现有使用。

### 3. 任务详情职责收口

任务详情页负责解释和复盘：

- 任务配置摘要。
- 能力点和目标关联。
- 近期执行情况。
- 未完成/推迟记录。
- 后续调整建议入口。

编辑动作统一回到现有任务编辑表单，不维护第二套编辑逻辑。

### 4. 上线边界

1.7.3 不做：

- 任务模板库。
- 自动推荐任务。
- 能力贡献事件。
- 强制所有任务必须关联能力点。
- 任务能力分自动计算。

## 三、学习计划改造

### 目标

计划页继续保持“周计划排程”定位，但补上家长最需要的判断：

- 本周计划重不重。
- 哪天任务偏多。
- 哪类任务覆盖不足。
- 当前安排是否适合孩子阶段。

### 1. 周计划摘要条

在计划页顶部或周视图上方增加摘要：

- 本周任务安排数。
- 预计总时长。
- 已完成数。
- 待完成数。
- 推迟/未完成数。
- 今日安排数。

### 2. 每日负荷提示

按天统计：

- 当日任务数。
- 当日预计时长。
- 是否偏重。

1.7.3 只用简单规则：

- `<= 45 分钟`：轻量。
- `46-90 分钟`：正常。
- `> 90 分钟`：偏重。

后续 1.8 再根据 L1-L5 阶段做动态阈值。

### 3. 计划与今日概览联动确认

修复或确认：

- 修改计划后今日概览能刷新。
- 移动/删除计划任务后今日任务不保留旧安排。
- 本周/下周切换时不串孩子、不串周。

### 4. 上线边界

1.7.3 不做：

- 自动生成最优计划。
- AI 排程。
- 批量跨周复制。
- 能力覆盖趋势。
- 阶段适配自动评分。

## 四、今日概览联动检查

任务和计划调整后，需要回归今日概览：

- 今日任务来自当前孩子的当前日期计划。
- 今日打卡后学习计划状态同步。
- 学习时长、完成率、专注时长仍按真实打卡统计。
- 任务卡片上的能力点显示与任务管理一致。

## 五、UI 标准

### 通用

- 控件高度统一为现有设计体系中的 `h-10` 或 `h-11`。
- 任务/计划页面不新增大面积装饰。
- 保持 `max-w-[1360px]` 主内容宽度。
- 卡片内文字不换乱、不溢出、不遮挡按钮。

### 任务页

- 卡片信息密度优先，不做营销式大卡。
- 批量状态和提醒用小徽标或提示条，不打断操作。

### 计划页

- 周计划表仍是主角。
- 摘要只辅助判断，不抢主视图。
- 状态图标保持 1.7.1 的方块/图标体系。

## 七、验收清单

### 自动检查

- `pnpm --dir frontend build`
- `pnpm --dir backend build`

### 手动回归

1. 登录。
2. 切换孩子。
3. 孩子编辑，检查 `学期与学习` 表单排列。
4. 任务管理：
   - 列表加载。
   - 新建任务。
   - 编辑任务。
   - 删除任务。
   - 缺能力点等提示正常。
5. 学习计划：
   - 当前周加载。
   - 切换周。
   - 移动任务。
   - 删除某日安排。
   - 查看周摘要和每日负荷提示。
6. 今日概览：
   - 今日任务与学习计划一致。
   - 打卡后计划状态更新。
7. `/api/health`
8. `/api/version`

## 八、上线原则

- 不引入数据库迁移，除非修复过程中发现生产必需问题。
- 不改变核心接口契约，优先在前端增加摘要和提示。
- 不把 1.8 能力模型大改提前混入 1.7.3。
- 如果构建或核心链路回归失败，不上线。
