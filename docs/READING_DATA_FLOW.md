# 图书馆与阅读数据闭环

## 当前数据来源

图书馆有三类核心数据：

- `books`：书籍基础信息，包括书名、ISBN、作者、出版社、分类、页数、字数、封面。
- `book_read_states`：某个孩子对某本书的阅读状态，包括在读、已读完和完成时间。
- `reading_logs`：阅读记录，包括阅读日期、页数、时长、备注、表现、阅读阶段。

## 导入清单的处理规则

如果 Excel 是“一书一行”，例如 `图书清单.xls`，系统按“已读书库”导入：

- 同步到书籍信息：书名、ISBN、作者、出版社、页数、字数。
- 根据 `阅读次数 > 0` 或 `最近一次阅读时间`，同步为当前孩子已读完。
- 如果有页数，会创建一条汇总阅读记录。
- 阅读记录日期使用 `最近一次阅读时间`。
- 没有阅读时长时，时长记为 0。
- 重复导入时，按 ISBN 或书名匹配已有书籍；阅读记录按“孩子 + 书 + 日期”避免重复创建。

这类清单不会拆成多条阅读明细，因为它没有每一次阅读的独立日期、页码和时长。

正式导入前，前端会先调用 `/api/library/import/preview` 做预检。预检只读取文件，不写数据库，用来确认：

- 字段识别结果。
- 将新增多少本书。
- 将匹配多少本已有书。
- 预计会创建多少条阅读记录。
- 预计会同步多少本已读状态。
- 是否缺少 `书名`、`ISBN` 等关键列。

## 精细阅读记录导入要求

如果要导入“西游记读了 16 次，每次都有日期、页码、时长”，Excel 必须是“一次阅读一行”：

```text
书名 / ISBN / 作者 / 出版社 / 阅读日期 / 开始页 / 结束页 / 阅读页数 / 阅读时长 / 阅读阶段 / 备注
```

这种格式才能生成多条 `reading_logs`。

## 仪表盘阅读能力计算口径

`/reading/stats` 负责给仪表盘的阅读能力页提供真实数据：

- 正在阅读：`book_read_states.status = reading`
- 本周记录：本周 `reading_logs` 数量
- 本月记录：本月 `reading_logs` 数量
- 所选范围记录：当前时间范围内 `reading_logs` 数量
- 书库总量：当前家庭已收录图书数
- 每日阅读完成趋势：当天有阅读记录记为完成
- 每日阅读时长：按当天 `reading_logs.minutes` 求和
- 阅读类型分布：按书籍分类统计；有时长时按时长占比，否则按记录数占比
- 阅读书籍 Top5：优先按阅读时长排序；时长相同按页数，再按记录次数
- 阅读能力雷达：持续度、阅读投入、类型广度、进度记录、复盘质量
