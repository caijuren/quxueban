# 趣学伴 1.9 三层准备度模型与诊断引擎设计方案

更新时间：2026-05-01

## 一、文档状态

状态：1.9 方案冻结前的详细设计稿。

本文件前半部分为 1.9 当前准口径；后半部分保留历史草案和讨论记录，用于追溯为什么从“五大核心 + 四个支撑模型”收敛为“三层结构 + 诊断引擎”。

1.9 的核心改动不是继续扩字段，而是把趣学伴从“任务和记录工具”升级为“升学准备度决策系统”的第一版。它需要同时解决三个问题：

- 家长能否一眼看懂孩子当前准备状态。
- 系统能否解释能力变化背后的原因。
- 后续小学三公版和初中版能否共用一套底层引擎，而不是做成两套割裂产品。

## 一点五、1.9 重要改动：三层准备度模型 + 诊断引擎

### 1. 产品定位

1.9 的新定位：

```text
交付层回答“现在够不够强”，认知层回答“为什么能变强”，稳定性层回答“能不能持续变强”。
```

这句话作为 1.9 能力模型和目标引擎的设计原则。产品不再只展示“孩子做了什么”，而是要逐步回答：

- 当前与目标校要求的差距在哪里。
- 这个差距是学科交付不足、认知方法不匹配，还是稳定性系统过载。
- 下一步应该冲刺、修复、减压，还是维持。

1.9 不追求一次性做出完整 AI 决策系统，但必须完成信息架构、数据口径和页面框架的迁移，为 1.9.x 和 2.0 的诊断能力留出稳定接口。

### 2. 三层结构

1.9 后，小学三公方向采用三层准备度模型。

| 层级 | 核心模块 | 核心指标 | 解决的问题 |
| :--- | :--- | :--- | :--- |
| 交付层 Delivery | 五大核心能力 | 英语、数理、大语文、项目成果、临场应变 | 能否拿到面谈邀请/录取，体现对外竞争力 |
| 认知层 Cognition | 思维能级 | 信息处理速度、规则内化、逻辑深度、批判思维、迁移应用 | 学习效率和能力天花板在哪里 |
| 稳定性层 Stability | 身心与执行状态 | 睡眠、运动、情绪、完成率、延期率、复盘频率 | 系统能否持续运转不崩盘 |

原“五大核心能力”不删除，而是归入交付层。原“四个执行支撑模型”不再作为同级展示结构，而是拆入稳定性层、数据证据和诊断引擎：

- 习惯与时间治理 -> 稳定性层的执行状态。
- 数据沉淀与反馈 -> 诊断引擎的数据证据。
- 身心与抗逆力 -> 稳定性层的身心状态。
- 家庭资源与信息 -> 环境扰动、目标配置和家长端策略建议。

### 3. 跨层因果逻辑

三层不是并列看板，而是存在因果传导：

- 稳定性层是带宽。睡眠不足、情绪波动、延期率高、复盘频率下降，会压缩认知层的信息处理速度、规则内化和迁移能力。
- 认知层是倍率。规则内化、迁移应用和逻辑深度越强，交付层的英语、数理、大语文增长速度越快。
- 交付层是输出。它是家长和目标校最容易看到的结果，但不应该被孤立解释。

产品上要避免只说“英语弱”“数理差”。更好的诊断表达是：

```text
英语阅读进阶缓慢。当前证据显示，近期复盘频率下降，复杂文本规则内化不稳定，可能共同压低了阅读进阶速度。建议本周减少新增阅读量，补充句法切片和复述任务。
```

### 4. 诊断引擎闭环

1.9 先做规则化诊断框架，后续再接 AI。

内部流程：

| 步骤 | 逻辑环节 | 核心动作 |
| :--- | :--- | :--- |
| Step 1 | 指标采集 | 从任务、阅读、打卡、复盘和测评中沉淀行为数据 |
| Step 2 | 趋势识别 | 计算 Velocity，识别能力增长动量 |
| Step 3 | 异常发现 | 捕捉斜率骤降、波动率激增、目标 Gap 扩大 |
| Step 4 | 环境剥离 | 检查考试周、生病、换老师、旅行等外部负载 |
| Step 5 | 跨层归因 | 判断是交付不足、认知阻力，还是稳定性过载 |
| Step 6 | 置信度校验 | 评估证据链是否完整，避免少量数据制造强结论 |
| Step 7 | 策略输出 | 给出冲刺、修复、减压、维持四类建议 |

1.9 首版只需要完成 Step 1、Step 2 的基础字段和 Step 7 的轻量展示；Step 3-6 做数据结构和规则说明，复杂计算进入 1.9.x。

### 5. 诊断引擎三道保险

为了避免教育数据误判，诊断引擎必须内置三道保险。

| 保险 | 目的 | 1.9 落地方式 |
| :--- | :--- | :--- |
| 置信度校验 Confidence | 防止少量数据导致过度判断 | 指标展示样本量、最近更新时间、数据来源数量；数据不足时显示“证据不足” |
| 趋势对冲 Velocity | 区分单次波动和长期趋势 | 先做 4 周/8 周趋势口径，复杂斜率计算后置 |
| 环境扰动 Context | 防止把外部压力误判为执行力下降 | 允许家长记录考试周、生病、旅行、换老师等外部负载 |

### 6. 首批异常诊断场景

1.9 文档中先定义规则脚本，产品首版可以只展示其中 4 个核心场景。

| 场景 | 触发条件 | 诊断方向 | 建议动作 | 版本 |
| :--- | :--- | :--- | :--- | :--- |
| 假性退步 | 单次测评分数下降，长期趋势稳定，稳定性无明显恶化，置信度低 | 偶然波动 | 继续观察，补充同类任务数据 | 1.9.x |
| 假性爆发 | 单次测评分数暴增，认知层未同步提升，置信度低 | 偶然超水平发挥 | 不立即跳级，保持难度继续验证 | 1.9.x |
| 高压透支 | 交付层表现好，但睡眠下降、延期率上升、情绪波动增加 | 短期超负荷 | 降低新增任务，保核心任务 | 1.9 |
| 认知阻力位 | 成长斜率趋平，但完成率、复盘和睡眠正常 | 方法或认知瓶颈 | 从刷题切到讲题、迁移、错因分类 | 1.9.x |
| 执行断裂 | Gap 扩大，完成率下降，延期率上升，复盘频率下降 | 执行系统失效 | 缩小任务颗粒度，恢复节奏 | 1.9 |
| 输入不足 | 英语/语文停滞，稳定性良好，但有效输入不足 | 素材量不够 | 增加高质量输入，不急着测评 | 1.9.x |
| 输出虚高 | 分数不错，但迁移、表达、开放题弱 | 应试掩盖真实能力 | 增加开放表达和项目化任务 | 1.9.x |
| 外部扰动 | 考试周、生病、旅行、家庭节奏异常 | 环境造成短期偏移 | 降低报警灵敏度，延后强结论 | 1.9 |

首批必须先做的四个：高压透支、执行断裂、外部扰动、数据不足提醒。它们最贴近 1.9 已有任务/打卡/目标数据，也最能保护家长和孩子不被错误结论带偏。

### 7. 认知层采集策略

认知层不应主要依赖家长观察。家长观察噪声高，且会增加负担。

1.9 的策略是：

```text
任务型测试和日常任务埋点为主，老师/系统评价为辅，家长观察只记录环境解释变量。
```

认知指标作为交付层任务的元标签存在。例如一道数理题不只记录对错，还记录：

```text
题目类型：新规则题
认知标签：规则内化、信息处理速度
首次正确：第 3 次
耗时：142 秒
是否看提示：是
提示后是否迁移成功：是
错因：漏读条件
复盘质量：能解释主要规则，但不能举反例
```

1.9 不需要完整实现自动埋点，但数据模型必须支持 `cognitiveTags`、`evidenceType`、`sampleCount`、`confidence` 等后续字段。

### 8. 认知层试点：先做数理逻辑

认知指标第一批试点建议放在数理逻辑，而不是英语。

原因：

- 数理过程数据更结构化，规则内化、迁移、速度和逻辑深度更容易观察。
- 英语受词汇量、材料熟悉度、语感和背景知识影响更大，早期噪声更高。
- 数理逻辑跑通后，可以把“任务元标签 -> 认知画像 -> 诊断建议”的闭环复用到英语和大语文。

首批认知指标：

| 认知指标 | 数理任务采集方式 | 可观察行为 |
| :--- | :--- | :--- |
| 规则内化 | 新规则题/新定义题 | 第几次尝试后能正确应用规则 |
| 迁移应用 | 变式题/跨场景题 | 能否把刚学规则迁移到新题型 |
| 信息处理速度 | 限时小题组 | 单位时间正确率、跳题率、回看率 |
| 逻辑深度 | 讲题/写解题路径 | 是否能说明条件、步骤、依据和反例 |

1.9 首版只需要在任务和记录里预留认知标签、错因、复盘质量和任务类型；自动评分进入 1.9.x。

### 9. 余力指数与机会成本

三公准备不是所有能力都无限加码，而是资源分配问题。1.9 需要引入“余力指数”的设计口径，首版可先不做复杂计算。

余力指数用于判断孩子还有没有空间承接新增任务：

- 睡眠、情绪、延期率、完成率稳定 -> 有余力。
- 睡眠下降、情绪波动、延期增加 -> 余力不足。
- 某一能力已超过目标水位且趋势稳定 -> 进入维持模式，释放资源给 Gap 更大的能力。

示例规则：

```text
IF 英语已高于目标校水位
AND 英语趋势稳定
AND 数理 Gap 较大
AND 稳定性层余力足够
THEN 英语进入维持模式，新增学习时间优先分配给数理逻辑
```

1.9 首版可在目标页和建议卡片里使用“维持/加码/修复/减压”的策略标签，不必先做精确时间分配算法。

### 10. 目标校适配权重

小学版不能只给一个泛化准备度分数。后续需要支持不同目标校偏好权重：

- 上实方向：提高数理逻辑、快速规则学习、空间想象、项目探究权重。
- 上外附中方向：提高英语结构化表达、大语文、人文素养、综合表达权重。
- 浦外方向：提高英语能力、综合表达、临场应变权重。

1.9 的处理方式：

- 先在数据结构上保留 `goalTemplate` 和 `weightProfile`。
- 页面上先展示“三公基础模板”。
- 上实/上外/浦外适配度指数进入 1.9.x，不进入 1.9 P0。

### 11. 小学版和初中版的技术抽象

后续初中版会与小学三公版有明显差异，但技术上可控，前提是 1.9 不把小学三公逻辑写死。

底层应抽象为：

```text
Stage：学段/升学阶段
Goal：目标路径，例如三公、中考、自招、国际方向
Layer：交付层、认知层、稳定性层
Module：英语、数理、大语文、物理、化学等
Metric：具体指标
Rule：诊断规则
Weight：权重配置
Recommendation：建议模板
```

小学三公版本：

```text
交付层 = 英语 + 数理 + 大语文 + 项目成果 + 临场应变
认知层 = 规则内化 + 迁移应用 + 信息处理速度 + 逻辑深度 + 批判思维
稳定性层 = 睡眠 + 运动 + 情绪 + 完成率 + 延期率 + 复盘频率
目标 = 三公基础 / 上实 / 上外附中 / 浦外
```

初中版本：

```text
交付层 = 语文 + 数学 + 英语 + 物理 + 化学 + 校内排名
认知层 = 知识掌握、错因识别、复习迁移、压轴题策略、表达论证
稳定性层 = 作息、情绪、考试压力、复习节奏、任务负荷
目标 = 中考 / 自招 / 特色高中 / 国际路线
```

因此，1.9 需要优先做配置化能力模型、目标模板和规则模板，不要把页面和字段只绑定到“三公”字样。

## 一点六、1.9 具体要做什么

### P0：必须进入 1.9

1. 信息架构改造

- 首页/能力模型页从“五大核心 + 四个支撑模型”改为“三层准备度”。
- 交付层、认知层、稳定性层都要有清晰解释。
- 原有任务、目标、阅读、打卡数据重新归类，不要求全部重建。

2. 能力模型配置化

- 支持小学三公基础模板。
- 支持初中“学科 + 通用学习能力”基础模板。
- 能力模型结构至少包含：层级、模块、能力点、观察指标、数据来源。
- 家庭编辑先轻量化，优先允许编辑目标和能力点文案；全层级复杂编辑后置。

3. 目标引擎升级

- 目标成为独立对象，不再只是任务字段。
- 目标可绑定能力点、周期、状态、成功标准。
- 任务必须能解释服务哪个目标和哪个能力点。
- 目标页展示目标进展和相关任务。

4. 任务模型整理

- 任务列表减少字段堆叠，只展示名称、类型、能力点、目标、今日状态。
- 任务详情承担完整解释：为什么做、服务哪个目标、如何记录、历史表现。
- 新建/编辑任务支持交付层/认知层/稳定性层归类。

5. 轻量诊断卡片

- 首页或能力模型页增加“本周建议”卡片。
- 第一版只输出四类策略：冲刺、修复、减压、维持。
- 必须支持“数据不足/证据不足”提示。
- 首批诊断只做高压透支、执行断裂、外部扰动、数据不足。

6. 稳定性层基础数据

- 完成率、延期率、复盘频率进入稳定性层。
- 睡眠、运动、情绪进入稳定性层。
- 增加外部负载记录：考试周、生病、旅行、换老师、家庭节奏异常。

7. 数理认知试点字段

- 任务或记录支持认知标签：规则内化、迁移应用、信息处理速度、逻辑深度。
- 支持错因、复盘质量、是否提示、耗时、尝试次数等轻量记录字段。
- 不做自动认知评分，只沉淀证据。

8. 兼容与迁移

- 旧任务的能力点和目标类型要能映射到新三层结构。
- 旧数据缺字段时页面不能崩，也不能展示强诊断。
- 旧的“成长能力模型/学科能力模型”文案需要同步替换为新口径。

### P1：可以进入 1.9，视工作量裁剪

- 简单成长斜率：4 周趋势、8 周趋势。
- 指标置信度：样本数量、数据来源数量、最近更新时间。
- 目标校模板字段：只做数据结构，不做完整适配度评分。
- 余力指数初版：用策略标签表达，不做复杂算法。
- 报告页展示三层摘要和目标进展。
- 今日概览展示“今日任务服务的目标/层级”。

### 暂不进入 1.9

- AI 自动诊断报告。
- 完整学校适配度分数。
- 完整认知自动评分。
- 复杂跨学科交叉验证。
- 完整错题系统。
- 大型仪表盘重构。
- 精细化目标校数据库。
- 自动生成个性化学习路径。

## 一点七、1.9 后续分步路线

### Step 0：1.8 收尾

目标：先保证当前系统稳定，不把 1.8 遗留问题带入 1.9。

- 完成 1.8 全链路回归。
- 修复任务详情、新建任务、今日概览、学习计划、钉钉推送阻塞问题。
- 发布 1.8。

### Step 1：1.9 设计冻结

目标：冻结三层结构和数据口径。

- 确认小学三公三层模型。
- 确认初中版抽象方式。
- 确认 P0/P1/后置范围。
- 确认旧字段迁移映射表。
- 确认首批诊断卡片和文案尺度。

### Step 2：1.9 数据模型与配置层

目标：先把底层抽象搭好。

- 新增或调整能力模型配置。
- 新增目标对象或增强目标字段。
- 建立目标、任务、能力点关系。
- 支持三层结构、目标模板、观察指标、数据来源。
- 做旧数据兼容和默认模板初始化。

### Step 3：1.9 页面信息架构

目标：让家长能看懂新模型。

- 能力模型页改为三层准备度展示。
- 目标页围绕能力和周期重构。
- 任务列表降低字段噪音。
- 任务详情解释能力、目标和记录方式。
- 首页增加轻量“本周建议”区域。

### Step 4：1.9 轻量诊断与稳定性层

目标：形成第一版“看数据 -> 给动作”的闭环。

- 接入完成率、延期率、复盘频率。
- 接入睡眠、运动、情绪和外部负载。
- 输出高压透支、执行断裂、外部扰动、数据不足提醒。
- 支持冲刺、修复、减压、维持四类策略建议。

### Step 5：1.9 数理认知试点

目标：验证认知层不是玄学。

- 为数理任务增加认知标签。
- 记录尝试次数、耗时、提示、错因、复盘质量。
- 页面先展示证据，不做复杂评分。
- 为 1.9.x 的认知阻力位规则积累数据。

### Step 6：1.9 回归发布

目标：确保小学和初中链路都能用。

- 回归小学三公模板。
- 回归初中基础模板。
- 回归旧任务兼容。
- 回归目标、任务、今日概览、学习计划、报告。
- 发布 1.9。

### Step 7：1.9.x 诊断增强

目标：把 1.9 的结构升级成更强的规则引擎。

- 增加假性退步、假性爆发。
- 增加认知阻力位。
- 增加输入不足、输出虚高。
- 增加置信度和成长斜率展示。
- 增加目标校适配权重基础版。
- 增加余力指数基础版。

### Step 8：2.0 AI 驱动

目标：在稳定规则和数据证据基础上接入 AI。

- AI 生成周报/月报草稿。
- AI 解释能力变化。
- AI 发现数据缺口。
- AI 推荐目标和任务草稿。
- AI 根据目标校模板生成策略建议。

约束：AI 不能绕过证据链，数据不足时必须明确说明。

## 一点八、工作量判断

从界面角度，这是中到大的信息架构改动，不是简单文案替换。但它不是全部页面重做，原有数据资产可以复用。

建议工作量预估：

| 版本 | 范围 | 预估 |
| :--- | :--- | :--- |
| 轻量版 | 三层展示、原指标重归类、简单建议卡片 | 1-2 周 |
| 标准 MVP | 三层模型、目标引擎、任务重构、稳定性层、轻量诊断、数理认知试点 | 3-6 周 |
| 完整专业版 | 跨学科验证、目标校权重、余力指数、AI 报告、复杂诊断 | 2-3 个月以上 |

推荐 1.9 做标准 MVP 的前半段：结构、目标、任务、稳定性和轻量诊断先打通；复杂诊断和 AI 后置。

## 一点九、当前方案价值判断

这套方案的价值不在于多几个图表，而在于把家庭教育中的高频错误决策系统性降低：

- 防止一次低分导致过度焦虑。
- 防止一次高分导致盲目跳级。
- 防止孩子已经过载时继续加压。
- 防止平台期时继续无效刷题。
- 防止家长只刷强项，忽略整体最优。
- 防止小学版和初中版后续技术割裂。

在上海家长场景里，这套理念属于高水平、强系统化路线。它不靠更鸡血建立价值，而是靠更冷静的诊断、更低的认知负担和更清晰的资源分配建立壁垒。

实现时要警惕“精致焦虑”：页面不要把所有指标都堆给家长，而要突出当前状态、核心风险、下一步动作和为什么这样建议。

本文件先定义 1.9 的问题边界、核心对象、待决策问题和推荐拆解顺序。后续每次讨论完一个模块，再把“待定”改成“已定”。

当前待讨论重点：

- 小学模型不是泛泛的成长能力模型，而是面向“三公/强校目标”的五年长期规划模型。
- “五个”优先指小学五年 L1-L5 五个规划级别，不是能力分类必须只有五类。
- 分类要按三公目标真实要求拆，例如学科能力、阅读语言、思维能力、体育健康、奖项项目、习惯执行等。
- 初中“学科能力模型”如何在通用学科能力基础上，支持目标校模板，例如交附嘉分方向。
- 目标和能力点如何解耦，避免目标只是任务字段的别名。
- 任务、计划、打卡、阅读记录如何回流到能力和目标。
- 1.9 做到基础闭环，哪些复杂能力放到 1.9.x 或 2.0。

## 二、1.9 一句话目标

候选表述：

```text
让任务、阅读、打卡和目标不再只是记录，而是沉淀成可解释的能力成长数据。
```

更具体一点：

```text
1.9 要把“孩子在做什么”升级为“这些行动正在服务哪些能力和目标，并能看到进展”。
```

这意味着 1.9 的重点不是继续加字段，而是把已有对象之间的关系理顺：

- 能力模型：孩子长期发展的地图。
- 目标：一段时间内要推进的方向。
- 任务：每天执行的行动。
- 记录：任务完成后的证据。
- 报告：基于证据解释能力和目标进展。

## 三、1.9 不直接写代码的原因

1.9 会影响多个核心模块：

- 孩子画像
- 能力模型
- 目标管理
- 任务管理
- 学习计划
- 今日概览
- 阅读中心
- 图书馆
- 报告和统计
- 后续 AI 建议

如果不先设计，容易出现这些问题：

- 任务字段继续膨胀，但没有清晰业务含义。
- 能力点、目标、任务标签互相重复。
- 小学和初中的模型只是文字不同，底层逻辑仍混在一起。
- 报告和 AI 后续拿不到可靠数据。
- 家长看到很多配置，却不知道怎么用。

## 四、核心对象初稿

### 1. 能力模型

能力模型回答：

```text
孩子长期要发展哪些能力？
```

能力模型不是任务清单，也不是短期目标。它更像一张长期地图。

初步结论：

- 能力模型不应该在“家长可理解”和“系统可计算”之间二选一。
- 正确拆法是“家长呈现层 + 系统计算层”。
- 家长呈现层使用能理解、能行动的语言，例如阅读积累、专注执行、错题复盘、复习效率。
- 系统计算层负责把任务、阅读、打卡和目标记录映射成统计指标，例如连续完成天数、阅读分钟数、错题复盘次数、延期率、质量评分、未完成原因。

示例：

```text
家长看到：阅读积累
系统计算：阅读天数、阅读时长、阅读页数、复述次数、家长评价、读后输出质量
```

待讨论：

- 能力模型是否采用四层结构：分类、一级能力、二级能力点、观察指标。
- 家庭全层级可编辑时，如何避免系统模板失真和数据口径混乱。
- 系统默认模型和家庭自定义模型如何共存。
- 小学和初中是否共用一套底层表，只用 `educationStage` 区分。

### 2. 目标

目标回答：

```text
接下来一段时间重点推进什么？
```

目标应该和能力点有关，但不是能力点本身。

示例：

- 能力点：阅读理解
- 目标：连续 4 周保持每天 20 分钟阅读，并完成每周 1 次复述

待讨论：

- 目标周期：周目标、月目标、学期目标是否都做。
- 目标状态：进行中、已完成、暂停、放弃是否足够。
- 目标是否必须绑定能力点。
- 一个目标是否可以绑定多个任务。
- 一个任务是否可以服务多个目标。

### 3. 任务

任务回答：

```text
每天具体做什么？
```

1.8 已经给任务补了阶段、学科、能力点、目标类型、时间块、记录方式等字段。1.9 要做的是整理这些字段的关系。

待讨论：

- 任务列表卡片到底展示哪些字段，哪些放详情页。
- 分配规则、频次、时间块、记录方式的展示优先级。
- 小学任务和初中任务是否使用不同摘要模板。

已定原则：

- 学习/成长任务必须绑定至少一个目标。
- 学习/成长任务必须绑定至少一个能力点。
- 如果一个任务没有目标，也不解决能力点，就说不清为什么要做。
- 生活、运动、情绪、社交类任务也要绑定目标和能力点，例如整理书包绑定“自主管理”能力点，运动绑定“运动健康”能力点。

待讨论：

- 一个任务是否允许绑定多个能力点。
- 一个任务是否允许绑定多个目标。
- 多能力点、多目标任务的回流权重如何处理。

### 4. 记录

记录回答：

```text
执行结果是什么，有什么证据？
```

记录不只是完成/未完成，还应该逐步支持：

- 完成状态
- 完成数量
- 完成质量
- 专注时长
- 未完成原因
- 家长观察
- 孩子反馈
- 阅读页数、时长、表现

待讨论：

- 1.9 是否只做记录结构梳理，不做掌握度计算。
- 哪些记录字段必须进入能力回流。
- 未完成原因是否作为目标调整依据。
- 阅读记录如何映射到能力点。

### 5. 报告

报告回答：

```text
这些行动说明孩子发生了什么变化？
```

1.9 不建议做复杂报告大改，但要为 1.9.x 和 2.0 留出数据口径。

待讨论：

- 1.9 是否只在报告里展示目标进展摘要。
- 是否先做规则说明，不做复杂评分。
- 报告是否显示“数据不足”，避免假装有结论。

## 五、1.9 方向与现有 1.8 能力标签的区别

1.8 已经让任务能写入阶段、能力点、目标类型、学科、时间块和记录方式。但 1.8 本质上还是“任务字段增强”，不是完整能力模型。

1.9 的目标是从“字段标签”升级为“能力-目标-任务-记录”的闭环。

### 1. 分类方式不同

现有 1.8：

- 能力点主要是任务表单里的下拉选项。
- 小学和初中已经分阶段，但模型仍偏扁平。
- 能力分类、任务类型、目标类型之间关系不够清晰。

1.9 方向：

- 能力模型有稳定层级。
- 小学模型增加目标模板和 L1-L5 五年规划级别。
- 分类用于组织三公/强校准备方向。
- 一级能力用于家长理解和页面组织。
- 二级能力点用于任务、目标和记录绑定。
- 观察指标用于解释数据从哪里来。
- 小学使用三公目标导向规划模型，初中使用学科能力模型。

建议结构：

```text
目标模板 -> L1-L5 级别 -> 分类 -> 一级能力 -> 二级能力点 -> 观察指标 -> 关联任务/记录
```

示例：

```text
阅读积累
  - 阅读持续性
    - 每周阅读天数
    - 阅读总时长
    - 中断次数
  - 阅读理解
    - 复述次数
    - 家长评价
    - 阅读表现记录
```

### 2. 指标口径不同

现有 1.8：

- 任务保存了能力点，但系统还没有清晰说明这个能力点如何被计算。
- 完成记录、阅读记录、专注记录和能力点之间没有稳定回流口径。
- 今日概览和钉钉更多是文案阶段化。

1.9 方向：

- 每个能力点都要逐步定义可观察指标。
- 指标不一定马上做复杂评分，但必须知道数据来源。
- 可以先做“数据证据”，再做“规则掌握度”。

指标分层建议：

- 基础指标：完成次数、完成率、连续天数、延期次数。
- 质量指标：家长评价、孩子反馈、完成质量、复盘质量。
- 过程指标：专注时长、阅读时长、任务耗时、未完成原因。
- 阶段指标：周目标进度、月目标进度、阶段目标达成情况。

### 3. 任务角色不同

现有 1.8：

- 任务是主要对象，能力点和目标类型是任务的附属字段。
- 任务列表里字段较多，容易出现展示堆叠。

1.9 方向：

- 任务是能力和目标的执行单元。
- 任务创建时必须说明服务哪个能力点。
- 任务详情要解释“这个任务为什么存在”。
- 任务列表只展示最关键摘要，更多解释放详情页。

任务展示建议：

- 列表页：名称、学科/类型、核心能力点、分配规则、今日相关状态。
- 详情页：完整字段、目标关联、记录方式、历史表现、能力回流说明。
- 计划页：优先展示时间安排、频次、当天执行状态。

### 4. 目标角色不同

现有 1.8：

- `targetType` 已存在，但更多用于任务标签。
- “关联目标”还是静态选项，不能形成真正目标管理。

1.9 方向：

- 目标是独立对象。
- 目标绑定能力点，任务服务目标。
- 目标有周期、状态、成功标准和复盘。
- 目标进展能进入报告。

示例：

```text
能力点：阅读理解
目标：连续 4 周保持每天 20 分钟阅读，并每周完成 1 次复述
任务：睡前阅读 20 分钟
记录：阅读时长、页数、复述质量、家长观察
```

### 5. 家庭配置方式不同

现有 1.8：

- 系统内置能力点。
- 家庭暂时不能形成自己的能力模型模板。

1.9 方向：

- 系统提供默认模板。
- 后续提供多个模板，例如通用成长模板、阅读强化模板、交附嘉分目标校模板等。
- 家长优先编辑目标，而不是直接改底层模型。
- 家庭可以从系统模板复制成自己的模板，再逐步个性化。

已定方向：

- 家庭可以全层级编辑能力模型，包括分类、一级能力、二级能力点和观察指标。
- 系统需要提供默认模板，家庭可以在模板上修改并保存为自己的模板。
- 交附嘉分模板不是设计参考，而是重要目标模板，家庭会在这个模板上继续修改。
- 目标允许家长自由编辑。

待讨论：

- 全层级编辑是否放入 1.9 P0，还是先做“选择模板 + 编辑目标”，全量模板编辑放入 1.9.x。
- 家庭模板和系统模板如何做版本管理。
- 家庭修改模板后，历史任务和历史记录是否保留旧模型版本。

### 6. 数据回流不同

现有 1.8：

- 任务完成了，但对能力和目标的影响还不明确。
- 阅读、打卡、计划之间的数据没有统一解释层。

1.9 方向：

- 任务完成记录回流到能力点和目标进展。
- 阅读记录回流到阅读相关能力点。
- 未完成原因回流到目标调整建议。
- 先做可解释证据，不急着做复杂分数。

## 六、小学三公目标导向模型：待讨论草案

### 1. 方向修正

已定方向：

- 小学不是做一个泛泛的“成长能力模型”，而是做面向“三公/强校目标”的长期规划模型。
- 系统要把小学五年纳入同一套规划体系，而不是只看当周任务和当月目标。
- L1-L5 对应小学五个规划级别，表示不同年级/阶段的要求逐步提高。
- 能力分类不限制为五类，要按三公目标实际需要拆。

一句话：

```text
用五年时间，把孩子面向三公所需的英语、数理逻辑、大语文、校内荣誉和软实力准备，持续沉淀成可规划、可执行、可回顾的数据，并用习惯时间、数据反馈、身心抗逆和家庭资源作为支撑系统。
```

### 2. 模型层级

已定方向：

```text
目标模板 -> L1-L5 级别 -> 分类 -> 一级能力 -> 二级能力点 -> 观察指标 -> 任务/记录
```

解释：

- 目标模板：例如三公基础模板、上外方向模板、上实方向模板、浦外方向模板、交附嘉分衔接模板等。
- L1-L5：小学五年规划级别，每个级别的要求不同。
- 分类：三公/强校准备需要关注的大方向，分为核心准入能力和执行支撑模型。
- 一级能力：分类下面可被家长理解的能力板块。
- 二级能力点：任务和目标真正绑定的最小能力单元。
- 观察指标：系统可以收集和解释的数据。

### 3. 三公目标基础分类

三公目标基础分类分两层。

第一层是五大核心准入能力，即三公选拔真正看的“标尺”：

- 英语能力：标化成绩、原版阅读、词汇量、听说表达、结构化输出。
- 数理逻辑能力：数学思维、竞赛认证、超前知识框架、快速解题和空间想象。
- 大语文与人文素养：古文国学、百科文史哲、中文阅读、表达写作。
- 简历基石：校内成绩、成长记录册、官方荣誉、核心奖项、项目成果。
- 软实力与临场应变：抗压专注、快速学习、现场推理、团队协作、领导力。

第二层是四个执行支撑模型，即保障五大准入能力能长期落地的“动力系统与轨道”：

- 习惯与时间治理：弹性时间窗、自主学习、任务启动、持续完成、复盘整理。
- 数据沉淀与反馈：测评结果、阅读进阶、错题复盘、差距分析、阶段复盘。
- 身心与抗逆力：体能储备、睡眠作息、情绪韧性、压力恢复、稳定输出。
- 家庭资源与信息：信息过滤、资源匹配、战略定力、家庭支持、路线选择。

说明：

- 五大核心准入能力是 Ability Model 的主体。
- 四个执行支撑模型不一定都作为同级能力评分，但必须进入目标、任务和复盘体系。
- 体育健康目前更适合放入“身心与抗逆力”支撑模型，同时在三公材料或活动要求中作为证据项。
- 奖项与项目暂时归入“简历基石”，后续可细分为官方荣誉、竞赛证书、项目作品、活动经历。

待讨论：

- 四个执行支撑模型是否也需要像五大准入能力一样做能力点和指标。
- 体育健康是否在三公模板中单独提升为核心分类。
- 家庭资源与信息主要服务家长端，是否进入孩子能力模型页面，还是进入规划/复盘页面。

### 4. L1-L5 级别含义

初步方向：

- L1：一年级，基础兴趣、学习习惯和语言输入建立期。
- L2：二年级，稳定执行、阅读积累和基础学科扩展期。
- L3：三年级，英语、数理、大语文和思维能力系统积累期。
- L4：四年级，目标校准备加速期，开始补短板、做项目和沉淀证据。
- L5：五年级，面向升学节点的综合呈现期，强调稳定输出、表达、材料和临场应变。

已定方向：

- L1-L5 先直接对应一年级到五年级。
- 这样最简单，也最容易计算、展示和规划。

待讨论：

- 如果未来加入六三制，L6 是否作为小学高年级，还是进入初中模型。
- 每个级别是否需要明确不同目标水位，例如基础线、优秀线、三公目标线。

关于“基础线 / 优秀线 / 三公目标线”的说明：

- 这不是新分类，也不是给孩子贴标签。
- 它用于做差距分析：当前水平离普通达标、较好表现、目标校要求分别差多少。
- 如果这个概念容易制造焦虑，可以先不在页面直接展示，只在系统内部用于规划建议。
- 命名可以再讨论，暂时不用“达标线 / 优秀线 / 目标校线”作为最终产品文案。

### 5. 分类与级别的关系

同一个分类在 L1-L5 的要求不一样。

示例：英语能力

- L1：建立英语听读兴趣和基础语音输入。
- L2：形成稳定听读节律，开始分级阅读积累。
- L3：词汇、阅读理解和口头表达进入系统积累。
- L4：面向原版阅读、标化测评和结构化表达补短板。
- L5：具备面谈表达、长文理解和目标校材料中的英语证据。

示例：数理逻辑能力

- L1：数学兴趣、基础计算和简单逻辑游戏。
- L2：计算稳定，开始接触规律、图形、应用题思维。
- L3：系统进入数学思维模块，如数论、几何、行程、组合的基础。
- L4：结合目标校要求做模块化提升和竞赛/测评准备。
- L5：强调综合应用、快速推理、稳定输出和临场解题。

### 6. 任务、目标和规划的关系

已定原则：

- 目标来自分类和 L1-L5 级别要求。
- 任务必须服务目标。
- 任务必须绑定能力点。
- 记录要能回流到能力点和级别进展。

示例：

```text
目标模板：三公基础模板
级别：L3
分类：英语能力
一级能力：原版阅读
二级能力点：分级阅读持续推进
目标：本月完成固定级别分级阅读，并每周做 1 次口头复述
任务：早读完成 RAZ 阅读和跟读
记录：阅读级别、阅读时长、词汇掌握、复述完整度、家长评价
```

### 7. 当前仍需补充的外部资料

三公方向需要进一步整理资料，避免模型只来自主观想象。

待补充：

- 上海市实验学校、上外附中、浦外等公开招生要求和特色项目。
- 面谈、综合素质、语言、思维、体育、项目类准备的真实案例。
- 家庭对“三公”的优先目标：更偏学科、语言、综合素质，还是项目证据。

说明：

- 公开招生简章只能作为输入之一。
- 系统模型要把外部要求转成可执行的能力、目标、任务和记录。

### 8. 三公模板与交附嘉分模板的关系

已定方向：

- 三公是小升初目标模板。
- 交附嘉分是初中阶段的中考/高中目标模板。
- 两者不是简单父子继承关系，而是同一孩子成长路径上的两个阶段目标。
- 三公达成后，不代表自动满足交附嘉分要求，但三公阶段沉淀的数据会成为进入初中后继续规划的基础。

关系可以理解为：

```text
小学阶段：三公基础模板 / 上外方向 / 上实方向 / 浦外方向
  -> 沉淀英语、数理、大语文、简历、软实力、执行支撑数据
初中阶段：交附嘉分衔接模板
  -> 在已有数据基础上转入学科、错题、复习、考试策略和综合素质规划
```

待讨论：

- 小学三公模板结束时，哪些数据要带入初中交附嘉分模板。
- 三公目标没有达成时，是否仍然可以进入交附嘉分衔接模板。
- 交附嘉分模板第一版是否先从初中学科、错题复盘、复习效率、体育健康和综合素质五块展开。

## 七、初中学科能力模型：待讨论草案

初中阶段关键词：

- 学科
- 知识点
- 错题
- 复习
- 效率
- 考试策略
- 阶段目标
- 自主学习

### 候选组织方式

方案 A：按学科组织

- 语文
- 数学
- 英语
- 物理
- 化学
- 生物
- 历史
- 地理
- 道法

优点：家长和孩子容易理解。  
风险：能力模型容易变成课程表，跨学科能力不明显。

方案 B：按能力组织

- 学科基础
- 知识掌握
- 错题复盘
- 复习效率
- 考试策略
- 思维逻辑
- 自主学习
- 时间管理

优点：更像能力模型。  
风险：具体到学科时不够直观。

方案 C：学科 + 通用学习能力双层

- 学科能力：语文、数学、英语、理化生、史地政。
- 学习能力：错题复盘、复习效率、考试策略、时间管理、自主学习。

优点：兼顾家长理解和系统计算。  
风险：实现复杂度更高，任务可能同时挂多个维度。

已定方向：

- 初中采用“学科 + 通用学习能力双层”。
- 学科层解决“学什么、哪门课强弱”的问题。
- 通用学习能力层解决“怎么学、为什么效率不稳定”的问题。
- 任务可以同时服务一个学科能力和一个通用学习能力。

### 目标校模板方向

已定方向：

- 以通用初中学科能力模型作为系统基础模板。
- 在基础模板上增加目标校模板，其中交附嘉分是重要模板。
- 目标校模板不是机械复制招生简章，而是把公开招生政策、学校特色、家长期望和孩子当前阶段结合起来，转成可执行的能力和目标模板。

交附嘉分方向不是普通设计参考，而是家庭会实际使用和修改的目标模板：

- 学科基础：语文、数学、英语、理科、人文。
- 学习能力：错题复盘、复习效率、考试策略、自主学习、时间管理。
- 综合素质：表达沟通、运动健康、活动参与、稳定心态。

待补充资料：

- 公开招生简章和特色项目要求。
- 家庭对交附嘉分方向的真实目标。
- 当前孩子的年级、强弱项、考试和活动数据。

待讨论：

- 初中学科是否拆到单科，还是先合并“理化生”和“史地政”。
- 错题系统没完整上线前，错题复盘如何轻量记录。
- 交附嘉分模板在 1.9 P0 做到什么深度：内置模板、可复制模板，还是先做文档模板。

## 八、目标引擎：待讨论草案

### 目标类型候选

延续 1.8 已有类型：

- 习惯过程型 `habit_process`
- 能力成长型 `skill_growth`
- 知识掌握型 `knowledge_mastery`

可能新增：

待讨论：

- 是否区分“目标”和“指标”。

### 目标字段候选

草案字段：

```ts
Goal {
  id: number
  childId: number
  title: string
  description?: string
  educationStage: 'primary' | 'middle'
  goalType: string
  abilityPointIds: number[]
  startDate: string
  endDate?: string

  reviewFrequency?: 'weekly' | 'monthly'

```

待讨论：

- 目标是否必须有开始和结束时间。
- 成功标准是结构化字段，还是先用文本。
- 是否需要目标优先级。
- 是否需要家长手动复盘目标。

## 十一、建议开发拆分

### 阶段 0：设计冻结

- 定小学三公目标导向模型的基础分类。
- 定 L1-L5 五年规划级别要求。
- 定每个小学分类下的一级能力。

- 定交附嘉分目标模板的第一版结构。
- 定目标和能力点关系。
- 定任务列表字段展示规则。
- 定 1.9 P0 数据模型。

### 阶段 1：能力模型基础

- 把系统默认能力模型结构化。
- 能力模型页展示新模型。
- 允许家庭级轻量编辑或先只做查看。

### 阶段 2：目标引擎基础

- 目标结构升级。

## 十二、暂不做

1.9 暂不做：

- AI 自动推荐。
- 完整知识图谱。
- 完整错题系统。
- 复杂能力掌握度评分。
- 大型仪表盘重构。
- 学校差距量化。
- 复杂周报/月报生成。
- 大版本依赖升级。

## 十三、下一次讨论建议

建议先讨论能力模型，因为它会决定目标、任务和报告。

讨论顺序：

1. 五大核心准入能力是否确认：英语、数理逻辑、大语文、简历基石、软实力临场。
2. 四个执行支撑模型是否确认：习惯时间、数据反馈、身心抗逆、家庭资源。
3. L1-L5 直接对应一到五年级后，每个年级在五大准入能力上的要求如何定义。
4. “基础线 / 优秀线 / 三公目标线”是否只作为内部差距分析口径。
5. 每个核心分类下第一批一级能力是什么。
6. 三公模板沉淀哪些数据，进入初中交附嘉分衔接模板。
7. 家庭全层级编辑放入 1.9 P0，还是 1.9.x。
