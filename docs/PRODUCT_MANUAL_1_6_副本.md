# 趣学伴产品手册 1.6

更新时间：2026-04-28

## 产品定位

趣学伴是一款面向家庭学习管理的家长端产品，帮助家长围绕孩子的日常学习、阅读、目标、能力成长和复盘报告建立一个可持续的管理闭环。

当前版本定位为 `1.6.2 业务闭环稳定版`。核心重点不是复杂 AI 自动化，而是把家庭学习管理中最重要的对象先串起来：

- 孩子档案
- 今日任务
- 学习计划
- 任务库
- 图书馆与阅读记录
- 能力模型
- 阶段目标
- 仪表盘
- 学习报告
- 成就
- 设置与通知

## 核心使用路径

推荐家长按以下顺序使用：

1. 在 `设置` 中维护孩子档案、家庭信息、头像和推送配置。
2. 在 `能力模型` 中查看当前年级的能力要求。
3. 在 `目标` 中建立阶段目标，并关联能力点和任务方向。
4. 在 `任务管理` 中维护可复用任务。
5. 在 `学习计划` 中把任务安排到具体日期。
6. 在 `今日概览` 中执行当天任务和打卡。
7. 在 `图书馆` 中维护书籍、阅读状态和阅读记录。
8. 在 `仪表盘`、`学习报告`、`成就` 中复盘孩子成长情况。

## 日常执行

### 今日概览

今日概览是家长每天打开产品后的主工作台。

主要功能：

- 查看当天学习任务。
- 查看今日完成情况。
- 执行任务打卡。
- 查看学习进度与提醒。
- 进入当天相关任务和计划。

当前规则：

- 今日概览承接学习计划中的当天任务。
- 学习计划调整后，今日概览应同步反映。
- 今日概览不再保留重复的“安排计划”入口，排程统一回到学习计划。

### 学习计划

学习计划用于把任务安排到具体日期，解决“什么时候做”的问题。

主要功能：

- 按周查看计划。
- 添加、移动、删除计划任务。
- 发布计划。
- 查看每天任务安排。
- 与今日概览联动。

适用场景：

- 周末为下一周安排任务。
- 临时调整某天任务量。
- 把任务管理中的任务落到具体日期。

### 任务管理

任务管理是家庭任务库，用于维护可复用的任务。

主要功能：

- 创建学习、阅读、习惯、运动等任务。
- 设置任务类型、难度、年级等级、能力分类和能力点。
- 将任务关联到目标。
- 查看任务状态和任务详情。
- 支持任务模板方向的后续扩展。

业务定位：

- 任务是目标和计划之间的执行单元。
- 目标回答“为什么做”。
- 计划回答“什么时候做”。
- 任务回答“具体做什么”。

### 图书馆

图书馆用于管理孩子的书籍、阅读状态和阅读记录。

主要功能：

- 添加和编辑书籍。
- 上传和展示书籍封面。
- 管理书名、作者、ISBN、出版社、分类、页数、字数。
- 记录阅读状态：未记录、在读中、已读完。
- 添加、查看和编辑阅读记录。
- 按孩子隔离图书和阅读数据。
- 导入 Excel 书单。
- 导入前预检：展示将新增、匹配、跳过、同步已读和创建阅读记录的预计结果。

导入规则：

- 一书一行的清单会同步书籍基础信息。
- 如果有阅读次数或最近阅读时间，会同步为当前孩子已读完。
- 如果有页数、日期、时长等字段，会尝试创建阅读记录。
- 重复导入时优先按 ISBN 匹配，其次按书名匹配。
- 多次阅读明细需要“一次阅读一行”的模板，当前版本仍作为后续增强。

## 成长规划

### 能力模型

能力模型用于定义孩子在不同年级阶段应该关注的能力结构。

当前能力模型按 L1-L5 展示：

- L1：一年级
- L2：二年级
- L3：三年级
- L4：四年级
- L5：五年级

主要能力分类：

- 英语能力
- 语文能力
- 数学能力
- 思维与认知
- 学习习惯
- 体育与健康

当前版本能力模型以展示和业务承接为主。英语能力中已补入 L1-L5 的阶段目标、听说读写水平、材料进度和核心任务。

后续计划：

- 支持能力点编辑。
- 支持孩子年级自动映射默认级别。
- 支持能力点与目标、任务、报告之间的数据回流。

### 目标

目标用于承接能力模型，把能力要求转成阶段性的家庭行动方向。

主要功能：

- 创建和查看阶段目标。
- 目标关联能力分类和能力点。
- 目标关联任务。
- 查看阶段进度。
- 记录复盘节奏和目标说明。

业务定位：

- 能力模型是标准和参考。
- 目标是当前阶段的重点选择。
- 任务是目标下面的执行动作。
- 报告用于复盘目标是否有效推进。

当前版本目标页已经加强了对能力模型的承接，但能力模型和目标的边界仍会继续优化。

## 复盘分析

### 仪表盘

仪表盘用于查看孩子整体学习和阅读状态。

主要功能：

- 查看本周、本月等时间范围的数据。
- 查看学习任务、阅读、趋势和分类。
- 查看阅读能力相关数据。
- 作为家长复盘时的总览入口。

当前阅读统计口径：

- 正在阅读：来自图书馆阅读状态。
- 本周记录、本月记录：来自阅读记录。
- 每日阅读时长：来自阅读记录中的分钟数。
- 阅读类型分布：来自图书馆书籍分类。
- 阅读书籍 Top5：优先按阅读时长，其次按阅读页数和记录次数。
- 阅读能力雷达：当前采用持续度、阅读投入、类型广度、进度记录、复盘质量等维度。

### 学习报告

学习报告用于周期性复盘。

主要功能：

- 查看阶段报告。
- 汇总任务完成、目标推进、阅读记录等信息。
- 为下一轮目标和计划调整提供依据。

后续计划：

- 增强规则化分析建议。
- 支持从报告生成目标或任务。
- AI 报告分析后移到后续版本。

### 成就

成就用于长期激励。

主要功能：

- 展示孩子获得的成就。
- 支持按任务完成、目标达成、阅读里程碑、连续打卡等行为扩展。

当前版本以展示和基础管理为主，积分商城和复杂奖励机制暂不进入 1.6。

## 部署与运维

当前推荐部署方式：

```bash
cd /home/ubuntu
./scripts/deploy-production.sh
./scripts/check-production.sh
```

部署脚本负责：

- 拉取最新代码。
- 安装依赖。
- 执行 Prisma generate 和 migrate deploy。
- 构建后端。
- 构建前端。
- 发布前端 dist 到 Nginx 目录。
- 重载 Nginx。
- 重启 PM2 后端服务。
- 校验 health 和 version。

检查脚本负责：

- 查看 Git 状态。
- 查看包管理器和版本。
- 查看后端 health/version。
- 查看 PM2 状态。
- 检查头像字段和上传路由。
- 检查前端 dist 和 Nginx 配置。

当前阶段不建议 main 分支 push 后全自动上线。更稳妥的流程是：

1. 本地提交并推送 GitHub。
2. 服务器手动执行部署脚本。
3. 执行检查脚本。
4. 手动登录正式环境验证关键流程。

GitHub Actions 自动部署建议后移到版本更稳定后，并优先采用手动触发，而不是 push main 自动上线。
